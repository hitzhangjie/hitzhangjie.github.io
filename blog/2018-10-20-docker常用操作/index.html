Docker 常用操作速查手册

Posted 2018-10-20 00:00 +0800 by ZhangJie ‐ 5 min read

分享:  

Docker 是一种容器技术，与虚拟机不同，容器更加轻量。Docker 简化了开发、运维人员的工作——Build, Ship, and Run Anywhere。

本文以实用为导向，按场景分类梳理 Docker 常用命令，方便日常速查。

1 镜像管理 ##

镜像（Image）是容器的模板，所有容器都基于镜像启动。

1.1 拉取与搜索 ##

# 搜索 Docker Hub 上的镜像
docker search centos
docker search nginx --filter=stars=100    # 按 star 数筛选

# 拉取镜像（默认 latest 标签）
docker pull centos

# 拉取指定版本的镜像
docker pull centos:7
docker pull nginx:1.24-alpine

# 从私有仓库拉取
docker pull registry.example.com/myapp:v1.0

1.2 查看镜像 ##

# 列出本地镜像
docker images
docker image ls

# 只显示仓库名和标签
docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"

# 查看悬空镜像（无标签的中间层镜像）
docker images -f dangling=true

# 查看镜像详细信息（分层、环境变量、入口点等）
docker inspect centos:7
docker history centos:7        # 查看镜像构建历史及每层大小

1.3 构建与打标签 ##

# 从 Dockerfile 构建镜像，-t 指定名称和标签
docker build -t myapp:v1.0 .

# 指定 Dockerfile 路径
docker build -f /path/to/Dockerfile -t myapp:v1.0 .

# 构建时不使用缓存
docker build --no-cache -t myapp:v1.0 .

# 给镜像打标签（相当于复制一个别名）
docker tag centos:7 mycentos:latest
docker tag myapp:v1.0 registry.example.com/myapp:v1.0

1.4 镜像导入导出 ##

# 将镜像保存为 tar 文件
docker save -o centos.tar centos:7
docker save centos:7 | gzip > centos.tar.gz    # 压缩保存

# 从 tar 文件加载镜像
docker load -i centos.tar
docker load < centos.tar.gz

# 导入/导出容器文件系统（会丢失历史和元数据）
docker export container_id -o container.tar
docker import container.tar myimage:latest

save/load 保留镜像层和历史，适合镜像迁移；export/import 只保留文件系统快照，体积更小。

1.5 删除镜像 ##

# 删除指定镜像
docker rmi centos:7
docker image rm centos:7

# 强制删除（即使有容器引用）
docker rmi -f centos:7

# 删除所有悬空镜像
docker image prune

# 删除所有未被容器引用的镜像
docker image prune -a

2 容器管理 ##

容器（Container）是镜像的运行实例，是日常使用最频繁的部分。

2.1 启动容器 ##

# 交互模式运行，并分配终端
docker run -it centos /bin/bash

# 后台运行
docker run -d --name mynginx nginx

# 挂载本地目录到容器
docker run -it -v ~/debugger:/root/debugger centos /bin/bash

# 端口映射（主机端口:容器端口）
docker run -d -p 8080:80 nginx

# 设置环境变量
docker run -d -e MYSQL_ROOT_PASSWORD=123456 --name mysql mysql:8

# 容器退出后自动删除
docker run --rm -it centos /bin/bash

# 限制资源
docker run -d --cpus=2 --memory=512m nginx

# 指定重启策略
docker run -d --restart=always nginx
docker run -d --restart=on-failure:3 nginx

2.2 查看容器 ##

# 列出运行中的容器
docker ps

# 列出所有容器（包括已停止的）
docker ps -a

# 自定义输出格式
docker ps --format "table {{.ID}}\t{{.Names}}\t{{.Status}}\t{{.Ports}}"

# 查看最新创建的容器（含已停止）
docker ps -l

# 只显示容器 ID
docker ps -q

# 查看容器详细信息（挂载、网络、环境变量等）
docker inspect container_name

# 查看容器内进程
docker top container_name

# 查看容器资源使用情况（CPU、内存、网络、磁盘）
docker stats
docker stats container_name

# 查看容器内文件系统变化（与镜像的差异）
docker diff container_name

2.3 进入容器与执行命令 ##

# 在运行中的容器内执行命令
docker exec container_name ls /app

# 以交互模式进入容器
docker exec -it container_name /bin/bash

# 以指定用户执行
docker exec -it -u root container_name /bin/bash

# 以指定工作目录执行
docker exec -it -w /app container_name /bin/bash

# 附加到容器的主进程（连接其 stdin/stdout/stderr）
docker attach container_name

attach 直接连到容器主进程，退出会导致容器停止；exec 启动新进程，退出不影响容器运行。

2.4 启停与生命周期 ##

# 停止容器（发送 SIGTERM，超时后发送 SIGKILL）
docker stop container_name

# 立即强制停止（发送 SIGKILL）
docker kill container_name

# 启动已停止的容器
docker start container_name

# 重启容器
docker restart container_name

# 暂停/恢复容器内所有进程
docker pause container_name
docker unpause container_name

# 阻塞等待容器停止，并返回退出码
docker wait container_name

2.5 日志与文件传输 ##

# 查看容器日志
docker logs container_name

# 实时跟踪日志
docker logs -f container_name

# 查看最近 N 行
docker logs --tail 100 container_name

# 按时间过滤
docker logs --since 2024-01-01T00:00:00 container_name
docker logs --until 2024-01-02T00:00:00 container_name

# 带上时间戳
docker logs -t container_name

# 从容器拷贝文件到主机
docker cp container_name:/app/log.txt ./log.txt

# 从主机拷贝文件到容器
docker cp ./config.yaml container_name:/app/config.yaml

2.6 提交与重命名 ##

# 将容器的当前状态保存为新镜像
docker commit -a "author" -m "installed gcc" container_id repo:tag

# 重命名容器
docker rename old_name new_name

2.7 删除容器 ##

# 删除已停止的容器
docker rm container_name

# 强制删除运行中的容器
docker rm -f container_name

# 删除所有已停止的容器
docker container prune

# 一次删除多个容器
docker rm container1 container2 container3

3 数据卷管理 ##

Volume 是 Docker 推荐的持久化数据方式，独立于容器生命周期。

# 创建数据卷
docker volume create myvolume

# 列出所有数据卷
docker volume ls

# 查看数据卷详细信息（挂载点在宿主机的位置）
docker volume inspect myvolume

# 删除未使用的数据卷
docker volume prune

# 删除指定数据卷
docker volume rm myvolume

# 使用数据卷启动容器
docker run -d -v myvolume:/data nginx

# 绑定挂载（bind mount）——直接挂载主机路径
docker run -d -v /host/path:/container/path nginx

# 只读挂载
docker run -d -v /host/path:/container/path:ro nginx

# tmpfs 挂载（仅内存，不写磁盘）
docker run -d --tmpfs /tmp nginx

4 网络管理 ##

Docker 提供多种网络驱动：bridge（默认）、host、overlay、none。

# 列出所有网络
docker network ls

# 创建自定义桥接网络（支持容器名 DNS 解析）
docker network create mynet

# 查看网络详细信息
docker network inspect mynet

# 将运行中的容器连接到网络
docker network connect mynet container_name

# 断开容器与网络的连接
docker network disconnect mynet container_name

# 使用指定网络启动容器
docker run -d --network=mynet nginx

# 使用 host 网络模式（容器与主机共享网络栈）
docker run -d --network=host nginx

# 删除网络
docker network rm mynet

# 删除所有未使用的网络
docker network prune

5 Docker Compose ##

Docker Compose 用于定义和运行多容器应用，通过 docker-compose.yml 描述服务。

# 启动所有服务（后台运行）
docker-compose up -d

# 重新构建并启动
docker-compose up -d --build

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f

# 停止所有服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v

# 重启单个服务
docker-compose restart nginx

# 进入服务容器
docker-compose exec nginx /bin/bash

# 执行一次性命令
docker-compose run --rm app python manage.py migrate

# 构建/拉取/推送服务镜像
docker-compose build
docker-compose pull
docker-compose push

6 登录与仓库 ##

# 登录 Docker Hub（需要 docker ID，非邮箱）
docker login

# 登录私有仓库
docker login registry.example.com

# 登出
docker logout

# 推送镜像到仓库
docker push myname/myimage:v1.0
docker push registry.example.com/myimage:v1.0

7 系统维护与清理 ##

# 查看 Docker 系统信息
docker info

# 查看 Docker 版本
docker version

# 查看磁盘占用
docker system df

# 查看各组件磁盘占用的详细信息
docker system df -v

# 一键清理所有未使用资源（镜像、容器、网络、构建缓存）
docker system prune

# 更彻底的清理（包含未使用的镜像）
docker system prune -a --volumes

8 常用组合场景 ##

8.1 停用并删除所有容器 ##

docker stop $(docker ps -q)
docker rm $(docker ps -aq)

8.2 批量删除退出码非 0 的容器 ##

docker rm $(docker ps -aq -f "exited=1")

8.3 查看容器 IP 地址 ##

docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' container_name

8.4 进入已停止容器的文件系统 ##

docker run --rm -it -v /:/host alpine chroot /host

8.5 将运行中的容器导出为新镜像 ##

docker commit container_id new_image:tag
docker save -o new_image.tar new_image:tag

总结 ##

本文以系统化的方式整理了 Docker 日常使用中的常用命令，覆盖了镜像管理、容器生命周期、数据卷、网络、Compose 及系统维护等核心场景。对于 Dockerfile 编写和镜像构建优化等更深入的话题，可参考本博客中的 Dockerfile 系统化实践 一文。
