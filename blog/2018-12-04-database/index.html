Go database/sql 使用指南

Posted 2018-12-04 14:32 +0800 by ZhangJie ‐ 4 min read

分享:  

背景 ##

Go 标准库 database/sql 提供了一套通用的 SQL 数据库访问接口。与 Java 生态中 MyBatis、Hibernate 这类重量级 ORM 框架不同，database/sql 的设计非常克制：它只定义了接口抽象，具体的数据库连接和 SQL 方言由各数据库驱动实现。这种"标准库定义接口 + 第三方提供驱动"的模式，让 Go 的数据库操作保持了简洁统一。

本文以 MySQL 为例，介绍 database/sql 的核心用法。

database/sql 的设计 ##

database/sql 的核心是 sql.DB 类型，它不是一个单一的数据库连接，而是一个连接池。你只需要创建一个 *sql.DB 实例，之后所有的 SQL 操作都通过它来完成，连接的管理、复用、回收由标准库自动处理。

要访问 MySQL，需要两样东西：

  1. database/sql — 标准库，定义通用接口
  2. go-sql-driver/mysql — MySQL 驱动，实现接口

驱动通过匿名导入（blank import）的方式注册：

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
)

_ "github.com/go-sql-driver/mysql" 会触发驱动的 init() 函数，将自身注册到 database/sql 中。之后你就可以通过 sql.Open("mysql", dsn) 来获取数据库连接了。

这种设计的好处是：切换数据库时，业务代码几乎不用变，只需要换驱动和连接字符串。

连接数据库 ##

func openDB(host, database, user, pwd string) (*sql.DB, error) {
    dsn := fmt.Sprintf("%s:%s@tcp(%s)/%s?charset=utf8mb4&parseTime=true", user, pwd, host, database)
    db, err := sql.Open("mysql", dsn)
    if err != nil {
        return nil, err
    }
    // 建议显式设置连接池参数
    db.SetMaxOpenConns(25)
    db.SetMaxIdleConns(5)
    return db, nil
}

几个注意点：

  • 编码：MySQL 的 utf8 并不是真正的 UTF-8（最多 3 字节），如果需要存储 emoji 等 4 字节字符，请使用 utf8mb4。
  • 连接池：sql.Open 并不会立即建立连接，只是初始化了连接池结构。实际的连接在首次执行 SQL 时才会懒加载建立。你可以通过 SetMaxOpenConns 和 SetMaxIdleConns 控制连接池大小。
  • 可用性验证：建议在启动时调用 db.Ping() 验证数据库是否可达。

CRUD 操作 ##

插入（INSERT） ##

func insertUser(db *sql.DB, name string, age int, sex int) error {
    result, err := db.Exec("INSERT INTO t_user(name, age, sex) VALUES(?, ?, ?)", name, age, sex)
    if err != nil {
        return fmt.Errorf("insert user: %w", err)
    }
    affected, err := result.RowsAffected()
    if err != nil {
        return err
    }
    if affected != 1 {
        return fmt.Errorf("expected 1 row affected, got %d", affected)
    }
    // 获取自增 ID
    id, _ := result.LastInsertId()
    log.Printf("inserted user with id: %d", id)
    return nil
}

查询（SELECT） ##

type User struct {
    ID   int
    Name string
    Age  int
    Sex  int
}

func queryUsers(db *sql.DB, name string) ([]*User, error) {
    rows, err := db.Query("SELECT id, name, age, sex FROM t_user WHERE name LIKE ?", "%"+name+"%")
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var users []*User
    for rows.Next() {
        u := &User{}
        if err := rows.Scan(&u.ID, &u.Name, &u.Age, &u.Sex); err != nil {
            return nil, err
        }
        users = append(users, u)
    }
    // 检查遍历过程中是否有错误
    if err := rows.Err(); err != nil {
        return nil, err
    }
    return users, nil
}

重要：rows.Next() 遍历结束后，务必调用 rows.Err() 检查遍历过程中是否有错误发生。另外 rows.Close() 必须调用，建议用 defer 确保资源释放。

单行查询可以用 QueryRow，代码更简洁：

func queryUserByID(db *sql.DB, id int) (*User, error) {
    u := &User{}
    err := db.QueryRow("SELECT id, name, age, sex FROM t_user WHERE id = ?", id).
        Scan(&u.ID, &u.Name, &u.Age, &u.Sex)
    if err == sql.ErrNoRows {
        return nil, nil // 没找到
    }
    if err != nil {
        return nil, err
    }
    return u, nil
}

更新（UPDATE） ##

func updateUserName(db *sql.DB, id int, newName string) error {
    result, err := db.Exec("UPDATE t_user SET name = ? WHERE id = ?", newName, id)
    if err != nil {
        return err
    }
    affected, _ := result.RowsAffected()
    if affected == 0 {
        return fmt.Errorf("user %d not found", id)
    }
    return nil
}

删除（DELETE） ##

func deleteUser(db *sql.DB, id int) error {
    _, err := db.Exec("DELETE FROM t_user WHERE id = ?", id)
    return err
}

事务 ##

对于需要原子性执行的多个操作，使用事务：

func transfer(db *sql.DB, fromID, toID int, amount float64) error {
    tx, err := db.Begin()
    if err != nil {
        return err
    }
    defer tx.Rollback() // 确保异常时回滚，已提交时 Rollback 是 no-op

    _, err = tx.Exec("UPDATE account SET balance = balance - ? WHERE id = ?", amount, fromID)
    if err != nil {
        return fmt.Errorf("debit: %w", err)
    }
    _, err = tx.Exec("UPDATE account SET balance = balance + ? WHERE id = ?", amount, toID)
    if err != nil {
        return fmt.Errorf("credit: %w", err)
    }
    return tx.Commit()
}

defer tx.Rollback() 是个好习惯：如果事务中途出错，Rollback 确保回滚；如果 Commit 成功，Rollback 是一个空操作，不会有副作用。

预处理语句（Prepared Statement） ##

对于需要重复执行的 SQL，可以使用预处理语句提高效率：

stmt, err := db.Prepare("INSERT INTO t_user(name, age, sex) VALUES(?, ?, ?)")
if err != nil {
    return err
}
defer stmt.Close()

for _, user := range users {
    _, err := stmt.Exec(user.Name, user.Age, user.Sex)
    if err != nil {
        return err
    }
}

完整示例 ##

下面是一个完整的可运行示例，演示了连接 MySQL、插入和查询的完整流程：

package main

import (
    "database/sql"
    "fmt"
    _ "github.com/go-sql-driver/mysql"
)

func openDB(host, database, user, pwd string) (*sql.DB, error) {
    dsn := fmt.Sprintf("%s:%s@tcp(%s)/%s?charset=utf8mb4", user, pwd, host, database)
    return sql.Open("mysql", dsn)
}

func main() {
    db, err := openDB("127.0.0.1:3306", "testdb", "root", "password")
    if err != nil {
        fmt.Println("open db:", err)
        return
    }
    defer db.Close()

    // INSERT
    result, err := db.Exec("INSERT INTO t_user(name, age, sex) VALUES(?, ?, ?)", "zhangjie", 28, 0)
    if err != nil {
        fmt.Println("insert:", err)
        return
    }
    affected, _ := result.RowsAffected()
    fmt.Printf("insert success, %d row(s) affected\n", affected)

    // SELECT
    rows, err := db.Query("SELECT id, name, age, sex FROM t_user")
    if err != nil {
        fmt.Println("query:", err)
        return
    }
    defer rows.Close()

    for rows.Next() {
        var id, age, sex int
        var name string
        if err := rows.Scan(&id, &name, &age, &sex); err != nil {
            fmt.Println("scan:", err)
            continue
        }
        fmt.Printf("id:%d name:%s age:%d sex:%d\n", id, name, age, sex)
    }
}

总结 ##

database/sql 的设计体现了 Go 一贯的哲学：接口最小化，功能正交化。它提供了连接池、参数化查询、事务等核心数据库操作的基础设施，但并不试图成为一个 ORM。这种设计让代码更可预测，也更容易调试。

更多高级用法（连接管理、超时控制、NULL 值处理等），建议参考 Go 官方提供的测试示例，那里涵盖了 database/sql 几乎所有的使用场景。
