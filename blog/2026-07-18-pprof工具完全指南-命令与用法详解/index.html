pprof 工具完全指南：命令与用法详解（译）

Posted 2026-07-18 10:30 +0800 by ZhangJie ‐ 3 min read

分享:  

原文：pprof/doc/README.md

pprof 是一个用于可视化与分析剖析数据的工具。它读取 profile.proto 格式的采样数据，生成文本和图形化报告，底层借助 dot 可视化包来渲染图形。

profile.proto 是一种 protobuf 格式，描述了一组调用栈及其符号化信息。通常它承载的是统计性采样的调用栈，也就是我们常说的 profiling 数据。剖析数据可以从本地文件读取，也可以通过 HTTP 远程拉取。多个同类型的 Profile 之间可以进行聚合或对比分析。

当采样数据中只包含机器地址时，pprof 可以借助操作系统自带的 binutils 工具（addr2line 和 nm）来完成符号化。

pprof 中的 Profile ##

每个 Profile 本质上就是一组符合 profile.proto 格式的采样（Sample）的合集。每个 Sample 对应位置层级（location hierarchy）中的一个点，并关联一个或多个数值以及一组标签（Label）。最常见的情况是，这些数据来自对某个程序的统计性采样。但 pprof 本身对 Profile 的语义并无硬性要求，所以其他用途也完全可能。

三种使用模式 ##

pprof 支持三种使用方式：

报告生成模式 ##

pprof <format> [options] source

pprof 按指定的输出格式生成报告后直接退出。

交互终端模式 ##

pprof [options] source

启动一个交互式 shell，用户可以在其中逐条输入命令。输入 help 获取在线帮助。

Web 界面模式 ##

pprof -http=[host]:[port] [options] source

在指定端口启动一个 HTTP 服务器，在浏览器中访问 http://<host>:<port>/ 即可使用图形界面进行交互式分析。

核心概念 ##

报告是基于从 Profile 采样中重建出的位置层级生成的。每个位置有两个关键度量值：

  • flat：该位置自身的值
  • cum（cumulative）：该位置自身加上所有后代位置的合计值

如果某个采样在同一条调用链中多次包含同一个位置（比如递归调用），该位置只会被计算一次。

常用选项 ##

pprof 提供了一系列选项来控制输出粒度、聚焦范围和排序方式：

排序与粒度 ##

选项说明
-flat（默认）按 flat 值降序排列
-cum按 cum 值降序排列
-functions（默认）按函数粒度聚合
-filefunctions按"文件名+函数名"粒度聚合
-files按文件粒度聚合
-lines按行粒度聚合
-addresses按指令地址粒度聚合
-noinlines将内联函数的指标归到其外层调用者。例如 pprof -list foo -noinlines profile.pb.gz 会生成一份带标注的源码列表，其中内联函数的指标被归到了执行内联的那一行

聚焦与过滤 ##

选项说明
-nodecount=N最多显示 N 条记录
-focus=regex只保留报告条目匹配 regex 的采样
-ignore=regex排除报告条目匹配 regex 的采样
-show_from=regex隐藏第一个匹配项以上的所有节点
-show=regex只显示匹配的条目
-hide=regex隐藏匹配的条目

样本值与单位 ##

每个采样可能包含多个数值。pprof 在生成报告时默认使用最后一个样本值。你可以通过 sample_index= 选项按编号（从 0 开始）或按名称来选择要分析的指标。

unit= 选项强制指定显示单位。例如 unit=sec 会让所有时间值以秒为单位输出。pprof 能识别绝大多数常见的时间和内存大小单位。

Tag：剖析的额外维度 ##

Tag（标签）由名称和值（数字或字符串）组成，数字值可以附带单位。Tag 相当于给采样数据增加了额外的维度，可以按这些维度对数据进行拆分观察。

Tag 过滤 ##

-tagfocus 选项根据标签值来筛选数据。语法是 -tagfocus=regex 或 -tagfocus=范围，会将数据范围限定在标签匹配的采样子集上。-tagignore 的作用相反，排除匹配的采样。如果两者同时指定且都匹配到同一个采样，该采样会被丢弃。

使用 -tagfocus=regex 时，regex 会与每个标签关联的每个值进行比对。指定 regex1,regex2 时，只有同时满足有标签值匹配 regex1 且另有一个标签值匹配 regex2 的采样才会被保留。

你也可以通过 -tagfocus=tagName=value 的写法来明确指定标签名。标签名必须完全匹配，值部分可以是 regex 或范围。使用 regex1,regex2 时，任一匹配即可。

几个示例：

# 只保留有数字标签值在 128KB~512KB 范围内的采样
-tagfocus 128kb:512kb

# 只保留名为 mytag 且值在 128KB~512KB 范围内的采样
-tagfocus mytag=128kb:512kb

# 保留有字符串标签值匹配 someregex 的采样
-tagfocus someregex

# 保留 mytag 值为 myvalue1 或 myvalue2 的采样
-tagfocus mytag=myvalue1,myvalue2

Tag 可视化 ##

-tags 选项列出所有可用标签及其取值，并展示每个标签各取值对应的采样值分布。

pprof 的调用图报告（如 -web 或 -dot）会自动把标签值可视化为"伪节点"。使用 -tagshow 和 -taghide 配合标签名的正则来显示或隐藏这些伪节点。

-tagroot 和 -tagleaf 选项用于生成伪栈帧。例如 -tagroot=mytag 会在调用树的根部插入代表该标签值的栈帧。-tagleaf=mytag 则在叶子位置插入。这在 -http 模式的树状视图中尤其好用。

文本报告 ##

命令/选项功能
-text逐行打印位置条目，包含 flat 和 cum 值
-tree以树形展示每个位置的前驱和后继
-peek=regex打印匹配位置的完整前驱和后继，不做裁剪
-traces逐行列出每个采样的完整调用栈

图形化报告 ##

pprof 用 DOT 格式生成报告，而后调用 graphviz 将其转换为最终的图片格式。位置层级被转换为图结构，节点代表报告条目。节点数量受 nodecount 控制的启发式算法裁剪。

命令/选项输出格式
-dot.dot 格式
-svgSVG 格式
-web生成 SVG 并在浏览器中打开
-png、-jpg、-gif、-pdf对应图片格式

如何读懂调用图 ##

调用图（Callgraph）中的每个视觉元素都有明确的含义：

节点颜色：

  • 红色：cum 值为较大正值（消耗多）
  • 绿色：cum 值为较大负值（常见于 Profile 对比，表示资源节省）
  • 灰色：cum 值接近零

节点字体大小：

  • 字体越大 → flat 绝对值越大
  • 字体越小 → flat 绝对值越小

边的粗细：

  • 粗边 → 该调用路径上消耗了更多资源
  • 细边 → 该调用路径上消耗较少资源

边的颜色：

  • 红色 → 正值大
  • 绿色 → 负值大
  • 灰色 → 接近零

虚线边：两个节点之间存在被裁剪掉的中间节点 实线边：一个节点直接调用另一个节点 "(inline)" 边标记：该调用被内联到了调用者内部

举个例子来帮助建立直觉：如果图中某个节点字体小且呈灰色，说明它的 flat 和 cum 值都不大，它不构成瓶颈。如果某个节点字体很大且呈深红色，说明该函数自身消耗了大量资源（flat 高），且它的子调用链也在大量消耗资源（cum 高）。如果字体小但颜色深红，意味着这个函数自身的消耗不大，但它调用的下游函数中有热点（flat 低 cum 高）——它是一个"调度者"，真正的瓶颈在它的某个子调用里。

一条粗而红的虚线边意味着沿这个方向有大量资源被消耗，但中间被裁剪掉了若干层调用。你需要顺着这个方向继续向下深挖。

源码注解 ##

pprof 能够生成带标注的源码报告。前提是源码或二进制文件在本地可及，且 Profile 包含足够的细节信息。

pprof 在当前工作目录及其所有祖先目录中搜索源码。二进制文件则在 $PPROF_BINARY_PATH 环境变量指定的路径下查找（默认值是 $HOME/pprof/binaries，Windows 上是 %USERPROFILE%\pprof\binaries）。pprof 先按文件名字查找，如果 Profile 中包含 linker 的 build ID，还会进一步到以 build ID 命名的子目录中去查找。

binutils 工具（addr2line、nm）被用来检查和反汇编二进制文件。pprof 在当前 $PATH 和 $PPROF_TOOLS 目录下寻找这些工具。

命令/选项功能
-list=regex对匹配的函数生成带标注的源码列表，每行源码附上 flat 和 cum 值
-disasm=regex对匹配的函数生成带标注的反汇编列表
-weblist=regex生成源码+汇编融合的带标注列表，在浏览器中打开

Profile 对比 ##

pprof 可以将两个兼容的 Profile 相减，从而直观地看出两次采样之间的差异——性能变好了还是变差了。

  • -diff_base=profile：用于比较两个 Profile。百分比计算以 diff base 的总采样数为基准
  • -base=profile：用于从一个累积型 Profile（如 Go 的 block profile）中减去一个更早的基准。百分比计算以源数据与基准数据的总差异为基准

-normalize 选项会在相减之前将源 Profile 的总值缩放到与基准 Profile 相同。这在判断两个 Profile 之间的相对差异时非常有用。

使用 -diff_base 时，某些条目可能出现负值。如果最终输出为 protobuf 格式，所有 diff base 的采样会被打上一个键为 pprof::base、值为 true 的标签，以便后续 pprof 处理时能区分源数据和基准数据。

使用 -base 处理累积型 Profile 时，所有值均为正数，百分比以总差异为基准。在一般情况下，某些条目可能为负值，此时百分比以地址级别聚合的绝对值的总和为基准。

远程拉取 Profile ##

pprof 可以从本地文件或 HTTP/HTTPS URL 读取 Profile。其原生格式是 gzip 压缩的 profile.proto，但也兼容 gperftools 的旧格式。

选项说明
-seconds=N请求持续 N 秒的 Profile（用于 CPU profile）
-timeout=NHTTP 请求超时时间（秒），默认由启发式算法决定

TLS 相关选项：

选项说明
-tls_cert=/path/to/certTLS 客户端证书文件
-tls_key=/path/to/keyTLS 私钥文件
-tls_ca=/path/to/caCA 证书文件

如果你需要在 URL 中将 https 替换为 https+insecure，pprof 会跳过服务端证书校验。

如果指定了多个源 Profile，pprof 会把它们全部读取进来并合并。这在聚合分布式任务中多个进程的 Profile 时十分有用。当然，待合并的 Profile 必须类型兼容——你不能把 CPU profile 和 heap profile 混在一起。

符号化 ##

pprof 可以为只包含地址信息的 Profile 补充符号信息。这对编译型语言来说非常重要，因为这类语言的 Profile 通常不包含函数名或源码位置。

  • 本地符号化：使用本地的 binutils 工具
  • 远程符号化：连接正在运行的目标进程，调用其符号化接口

符号化默认开启，由 -symbolize 选项控制：

选项值行为
-symbolize=none禁用所有符号化
-symbolize=local仅使用本地二进制进行符号化
-symbolize=remote仅通过目标进程的符号化 handler 进行

本地符号化时，二进制文件按 Profile 中记录的路���查找，其次在 $PPROF_BINARY_PATH 中查找。你也可以把主二进制的路径作为 pprof 的第一个参数传入：

pprof /path/to/binary profile.pb.gz

默认情况下，pprof 会对 C++ 符号进行反修饰（demangle）和简化，积极丢弃模板和函数参数信息。这由 -symbolize=demangle 控制：

选项值行为
-symbolize=demangle=none保留原始修饰名（如有）
-symbolize=demangle=full完整反修饰名，不做简化
-symbolize=demangle=templates反修饰，去掉函数参数但保留模板参数

Web 界面详解 ##

当通过 -http=[host]:[port] 启动后，pprof 会开启一个 Web 服务器并自动在浏览器中打开页面。这个界面支持对 Profile 数据进行多种格式的交互式浏览。

View 菜单 ##

顶部菜单栏中的 View 菜单用于在以下视图之间切换：

Graph（调用图） ##

默认视图。以函数为节点，以调用边连接调用者和被调用者。用户可以拖拽平移，滚动滚轮或使用触摸手势缩放。边上标注的数字表示从该调用路径消耗的资源量——例如从 FormatPack 到 FormatUntyped 的边上标着 5.72s，意味着从 FormatPack 调用到 FormatUntyped（及其子调用）总共消耗了 5.72 秒。

Flame Graph（火焰图） ##

火焰图以极其紧凑的方式展示调用关系。每个方框代表一个栈帧。调用者在上，被调用者紧挨其下。方框宽度与该栈帧在采样中出现的频率成正比。子节点按宽度从左到右降序排列。

向上钻取（查看调用者）：传统火焰图是自上而下的。pprof 的火焰图对此做了扩展——点击某个函数，可以同时展示以该函数为终点的调用栈。也就是说，你既能看到"这个函数调用了谁"，也能看到"谁调用了这个函数"。

Diff 模式：当使用 --diff_base 时，方框宽度正比于该子树中的增减量之和。红色阴影表示净增，绿色阴影表示净减。

内联展示：调用者和被调用者之间如果没有水平分隔线，说明这段调用被内联了。例如 X 调用 Y 调用 Z，如果 Y→Z 被内联了，X 和 Y 之间有分隔线，但 Y 和 Z 之间没有。

Annotated Source Code（源码注解） ##

在火焰图中右键点击某个方框，选择"Show source in new tab"，会得到一份带标注的源码视图。每一行源码附带两个数字：第一个不包括从该行发起的子调用耗时，第二个包括了这些耗时。

点击某一行可以展开显示内联的函数调用和对应的汇编代码。汇编指令以绿色显示。内联函数的源码以蓝色显示，并按内联层级缩进。

Disassembly（反汇编） ##

从 View 菜单中选择 “Disassemble” 即可看到纯指令顺序的反汇编视图。需要注意的是：除非你只聚焦到少数几个函数，否则反汇编视图可能会非常慢且数据量巨大。

Top Functions（热点函数） ##

一张表格，展示 flat 和 cum 两个指标排名靠前的函数。flat 指函数自身消耗的采样点数量，cum 指函数及其所有子调用消耗的采样点数量，二者均有绝对值和百分比。默认按 flat 降序排列，点击 Cum 表头可以切换为按 cum 排序。

Peek（速览） ##

以简洁的文本格式展示每个函数的调用者和被调用者。通常情况下，火焰图能更直观地传达同样的信息。

Config 菜单 ##

Config 菜单允许将当前的聚焦和隐藏设置保存为命名配置，以便后续快速切换。

  • Save as …：保存当前设置为一个命名配置
  • Default：恢复到默认状态（清除所有过滤条件）

已保存的配置作为菜单项出现，选中即可应用。当前生效的配置前有 ✓ 标记。点击配置名称右侧的 🗙 可以删除该配置。
