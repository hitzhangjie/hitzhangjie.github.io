转盘算法

Posted 2018-06-29 22:00 +0800 by ZhangJie ‐ 3 min read

分享:  

背景 #

在实际开发中，我们经常会遇到一类问题：从一组候选项中按权重随机选出一个。典型场景包括：

  • 抽奖活动：奖池中有多个奖品，一等奖中奖概率 1%，二等奖 5%，三等奖 20%，谢谢参与 74%，需要保证每次抽奖的结果符合预设的概率分布。
  • 流量分发：A/B 测试中需要将流量按比例分配到不同的实验组，比如对照组 50%、实验组 A 30%、实验组 B 20%。
  • 负载均衡：加权轮询（Weighted Round Robin）中，需要按服务器权重分配请求。
  • 遗传算法：在选择算子（Selection）阶段，根据个体适应度（fitness）按比例选择进入下一代的个体。

这些场景的共同特征是：每个候选项有一个权重，权重越高被选中的概率越大，且每次选择的结果需要不失随机性。

转盘算法（Roulette Wheel Selection）就是解决这类问题的经典方案。

算法原理 #

数学描述 ##

设有 $n$ 个候选项，每个候选项 $i$ 的权重为 $w_i$（$w_i > 0$），记总权重 $S = \sum_{i=1}^{n} w_i$。

那么候选项 $i$ 在一次独立抽样中被选中的概率为：

$$P(i) = \frac{w_i}{S}$$

这个公式的含义很直观：每个候选项被选中的概率，等于它的权重占所有候选项权重之和的比例。

转盘模型 ##

这个数学模型可以很形象地类比为一个转盘：

  • 将总权重 $S$ 看作转盘的周长。
  • 每个候选项 $i$ 在转盘上占据一段弧长为 $w_i$ 的扇形区域。
  • 转动指针，指针停在哪里，对应的候选项就被选中。

因为指针落在每个扇形区域的概率与该区域的弧长（即 $w_i$）成正比，所以恰好满足 $P(i) = w_i / S$。

算法步骤 ##

  1. 计算所有候选项的权重之和 $S$。
  2. 在 $[0, S)$ 区间内生成一个均匀分布的随机数 $r$。
  3. 遍历候选项，维护一个累加和 $sum$，每遍历到一个候选项 $i$，执行 $sum \mathrel{+}= w_i$。
  4. 当 $sum > r$ 时，当前候选项 $i$ 即为选中的结果。

这个步骤等价于：在转盘上随机投下一个点，看它落在哪个扇形区域中。

Go 语言实现 #

基础实现 ##

// RouletteItem 表示转盘中的一个候选项
type RouletteItem struct {
    Value  interface{} // 候选项的值，可以是任意类型
    Weight int         // 权重，正整数
}

// RouletteSelect 根据权重随机选择一个候选项
// 时间复杂度 O(n)，空间复杂度 O(1)
func RouletteSelect(items []RouletteItem) (interface{}, error) {
    if len(items) == 0 {
        return nil, errors.New("items must not be empty")
    }

    // 1. 计算总权重
    totalWeight := 0
    for _, item := range items {
        if item.Weight <= 0 {
            return nil, errors.New("weight must be positive")
        }
        totalWeight += item.Weight
    }

    // 2. 生成 [0, totalWeight) 范围内的随机数
    r := rand.Intn(totalWeight)

    // 3. 线性扫描，找到随机数落在的区间
    sum := 0
    for _, item := range items {
        sum += item.Weight
        if sum > r {
            return item.Value, nil
        }
    }

    // 理论上不会执行到这里
    return nil, errors.New("unreachable")
}

使用示例 ##

func main() {
    rand.Seed(time.Now().UnixNano())

    items := []RouletteItem{
        {Value: "一等奖-iPhone", Weight: 1},
        {Value: "二等奖-耳机", Weight: 5},
        {Value: "三等奖-优惠券", Weight: 20},
        {Value: "谢谢参与", Weight: 74},
    }

    // 模拟 10000 次抽奖，统计结果
    counts := make(map[string]int)
    for i := 0; i < 10000; i++ {
        result, _ := RouletteSelect(items)
        counts[result.(string)]++
    }

    for _, item := range items {
        v := item.Value.(string)
        actualRate := float64(counts[v]) / 10000.0 * 100
        expectRate := float64(item.Weight) / 100.0 * 100
        fmt.Printf("%s: 实际 %.2f%%, 期望 %.2f%%\n", v, actualRate, expectRate)
    }
}

运行结果示例：

一等奖-iPhone: 实际 1.12%, 期望 1.00%
二等奖-耳机: 实际 4.89%, 期望 5.00%
三等奖-优惠券: 实际 19.96%, 期望 20.00%
谢谢参与: 实际 74.03%, 期望 74.00%

可以看到，当抽样次数足够多时，实际中奖概率会趋近于预设的权重比例。

二分查找优化 ##

基础实现需要线性扫描所有候选项，时间复杂度为 $O(n)$。当候选项数量较大（如数千个商品需要按权重曝光）时，可以使用前缀和 + 二分查找将时间复杂度优化到 $O(\log n)$。

// RouletteFast 使用二分查找优化的转盘选择
// 预处理 O(n)（计算前缀和），每次查询 O(log n)
type RouletteFast struct {
    prefixSums []int        // 前缀和数组
    values     []interface{}
}

func NewRouletteFast(items []RouletteItem) *RouletteFast {
    prefixSums := make([]int, len(items))
    values := make([]interface{}, len(items))

    sum := 0
    for i, item := range items {
        sum += item.Weight
        prefixSums[i] = sum
        values[i] = item.Value
    }

    return &RouletteFast{
        prefixSums: prefixSums,
        values:     values,
    }
}

func (rf *RouletteFast) Select() interface{} {
    totalWeight := rf.prefixSums[len(rf.prefixSums)-1]
    r := rand.Intn(totalWeight)

    // sort.Search 等价于二分查找第一个 > r 的位置
    idx := sort.Search(len(rf.prefixSums), func(i int) bool {
        return rf.prefixSums[i] > r
    })

    return rf.values[idx]
}
实现方式时间复杂度适用场景
线性扫描$O(n)$ 每次候选项较少（< 100）
前缀和 + 二分$O(n)$ 预处理，$O(\log n)$ 每次候选项较多、频繁查询

变体场景 #

不放回抽样（Sampling Without Replacement） ##

在实际的抽奖活动中，常常是抽走一个奖品就少一个。比如用户抽中一等奖后，一等奖的总库存减 1，后续再抽时需要在剩余奖品中重新计算概率分布。

// RouletteSampleWithoutReplacement 不放回地依次抽出 k 个候选项
// 每次抽出后将该项从列表中移除，用剩余项重建转盘
func RouletteSampleWithoutReplacement(items []RouletteItem, k int) []interface{} {
    // 深拷贝，避免修改原切片
    pool := make([]RouletteItem, len(items))
    copy(pool, items)

    results := make([]interface{}, 0, k)
    for i := 0; i < k && len(pool) > 0; i++ {
        result, _ := RouletteSelect(pool)

        // 找到并移除已抽中的项
        for j, item := range pool {
            if item.Value == result {
                pool = append(pool[:j], pool[j+1:]...)
                break
            }
        }

        results = append(results, result)
    }
    return results
}

每轮抽选后，剩余奖品构成一个新的小转盘，各项的概率重新归一化为 $w_i / \sum_{j \in \text{pool}} w_j$。

保底机制（Pity System） ##

很多抽奖系统会引入保底机制：如果连续 N 次没抽到大奖，第 N+1 次必定抽中。这本质上是动态调整权重。

一种简单的实现思路：

type PityRoulette struct {
    items       []RouletteItem
    pityCounter int // 连续未抽中大奖的次数
    pityLimit   int // 保底触发阈值
    rareIndex   int // 大奖在 items 中的索引
}

func (pr *PityRoulette) Select() interface{} {
    pr.pityCounter++

    // 触发保底：直接返回大奖
    if pr.pityCounter >= pr.pityLimit {
        pr.pityCounter = 0
        return pr.items[pr.rareIndex].Value
    }

    result, _ := RouletteSelect(pr.items)

    // 如果抽中了大奖，重置计数器
    if result == pr.items[pr.rareIndex].Value {
        pr.pityCounter = 0
    }

    return result
}

更精细的做法是让大奖权重随未中次数递增：$w_{\text{rare}}’ = w_{\text{rare}} \times (1 + \alpha \times \text{pityCounter})$，这样概率平滑上升，而非在第 N 次时突变。

权重动态更新 ##

在流量分发场景中，可能需要根据实时反馈调整权重。如果权重频繁变更，可以考虑使用线段树或树状数组（Fenwick Tree）来维护前缀和，将更新和查询都优化到 $O(\log n)$，实现更高效的动态转盘。

总结 #

转盘算法是一种简洁而高效的加权随机选择算法，核心思想是将权重映射为概率空间中的区间长度，通过均匀随机数落到哪个区间来决定选中谁。

它的优势在于：

  • 原理直观，易于理解和实现。
  • 不失随机性，在大样本下精确符合预设的概率分布。
  • 可扩展性强，能与不放回抽样、保底机制、动态权重等需求组合。

在实际工程中，需要根据候选项的数量和查询频率选择合适的实现方式——候选项少时用线性扫描即可，候选项多且查询频繁时则用前缀和 + 二分查找优化。

此外，转盘算法还可以与蓄水池抽样（Reservoir Sampling）等算法结合，应对更复杂的实时数据流场景，但那就是另一个话题了。
