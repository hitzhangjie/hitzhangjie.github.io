为什么 vscode-go 无法正确处理嵌套子测试

Posted 2026-03-26 00:00 +0800 by ZhangJie ‐ 2 min read

分享:  

问题描述 ##

Go 测试文件中，常用 t.Run 来组织子测试。vscode-go 的 CodeLens 功能会为每个 t.Run 生成对应的 go test -run 命令。但当嵌套超过 2 层时，生成的命令是错误的：

  • 实际生成：go test -run=^TestXXX$/^t2$
  • 正确应为：go test -run=^TestXXX$/^t1$/^t2$

中间的层级 t1 被漏掉了。

问题定位 ##

Bug 在 extension/src/goRunTestCodelens.ts 第 112-153 行：

const simpleRunRegex = /t.Run\("([^"]+)",/;

for (let i = f.range.start.line; i < f.range.end.line; i++) {
    const line = document.lineAt(i);
    const simpleMatch = line.text.match(simpleRunRegex);

    // BUG: this does not handle nested subtests. This should
    // be solved once codelens is handled by gopls and not by
    // vscode.
    if (simpleMatch) {
        const subTestName = simpleMatch[1];
        // ... 只记录了名称，不知道嵌套层级
    }
}

代码里甚至有明确的 BUG 注释，承认这个问题。

根本原因 ##

当前实现是逐行正则匹配，没有层级概念。 遍历测试函数体的每一行，用正则匹配 t.Run("name",，找到后只记录 name，但不知道这一行处于哪个嵌套层级。因此 t1 和 t2 被平等对待，都只关联到顶层的 TestXXX。

为什么不从 gopls 获取嵌套结构？ ##

gopls 的 DocumentSymbol 不包含 t.Run ##

vscode-go 通过 LSP 向 gopls 请求 DocumentSymbol。这个接口返回的是声明（函数、变量、类型等），而 t.Run(...) 是函数调用，不是声明，所以 gopls 不会返回它。

Test Explorer 是怎么做到的？ ##

vscode-go 的 Test Explorer 能正确展示嵌套子测试，但它用的是动态发现：执行测试后解析 go test -json 的输出，从测试事件中动态创建子测试节点。但 CodeLens 是"显示在代码旁边的按钮"，需要在用户打开文件时立即生成，不能等到测试运行后。

修复思路 ##

既然 gopls 不提供嵌套信息，只能在 vscode-go 层面做轻量的静态分析。

方案：轻量括号追踪解析器 ##

不需要完整的 Go AST 解析器，只需要一个轻量的字符扫描器：

  1. 逐字符扫描测试函数体
  2. 跳过字符串和注释（避免误匹配）
  3. 遇到 t.Run("name", 时，记录当前嵌套深度和行号
  4. 记录每个 t.Run 调用的左花括号 { 和配对的右花括号 } 位置，确定子函数的作用域范围
  5. 通过作用域包含关系建立父子树
  6. 对每个 t.Run 节点，计算从根到该节点的完整路径

需要修改的文件 ##

文件操作说明
extension/src/subTestParser.ts新建实现括号追踪解析器
extension/src/goRunTestCodelens.ts修改替换简单正则循环
extension/test/unit/subTestParser.test.ts新建解析器单元测试

局限性 ##

修复后仍有以下已知局限：

  1. 动态测试名称无法检测：t.Run(varName, ...) 或 t.Run(fmt.Sprintf(...), ...) 无法静态分析
  2. 注释中的 t.Run 需要小心：解析器需要正确跳过注释

总结 ##

问题原因
嵌套子测试 CodeLens 生成错误逐行正则匹配，无法感知嵌套层级
gopls 无法提供帮助LSP DocumentSymbol 只描述声明，不包含函数调用
Test Explorer 能正确处理动态解析测试运行输出，非静态分析
修复方向在 vscode-go 层实现轻量括号追踪解析器
修复复杂度不高，不需要完整 AST，只需括号匹配+字符串/注释跳过
