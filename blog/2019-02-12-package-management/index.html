Linux 包管理：deb/apt 与 rpm/yum/dnf

Posted 2000-01-01 00:00 +0800 by ZhangJie ‐ 1 min read

分享:  

背景 #

不同 Linux 发行版采用不同的软件包格式和包管理工具，了解这些差异对于跨发行版工作非常重要。同时，选择合适的软件源镜像可以显著加速软件包的下载和安装。

正文 #

两大体系 ##

Linux 发行版的包管理大致分为两个阵营：

体系代表发行版包格式底层工具高层工具
Debian 系Debian, Ubuntu, Mint.debdpkgapt, apt-get
RHEL 系RHEL, CentOS, Fedora.rpmrpmyum (RHEL 7), dnf (RHEL 8+, Fedora)
  • dpkg 和 rpm 是底层工具，直接操作包文件，不自动处理依赖。
  • apt 和 yum/dnf 是高层工具，自动解析依赖关系并从远程仓库下载安装。

软件源镜像 ##

Linux 软件源中维护了一系列软件安装包列表。为了加速不同地域用户的访问，存在大量软件源镜像。其中清华大学开源软件镜像站覆盖面非常广，远超 Linux 发行版镜像的范畴，还维护了 BSD、Homebrew、ELK 等其他软件包。

CentOS 换源 ##

参考 CentOS 更换软件源帮助，替换 /etc/yum.repos.d/ 下的 repo 文件即可。

Ubuntu 换源 ##

参考 Ubuntu 更换软件源帮助，编辑 /etc/apt/sources.list 替换为清华镜像地址，然后执行 apt update。

总结 #

  • Debian 系用 apt + .deb，RHEL 系用 yum/dnf + .rpm。
  • 换国内镜像源能显著提升包下载速度，清华 tuna 镜像站是很好的选择。
  • 换源后记得刷新包索引（apt update 或 yum makecache）。
