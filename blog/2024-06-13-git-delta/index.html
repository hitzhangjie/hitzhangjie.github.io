Git Delta：在终端里 Side-by-Side 看 diff

Posted 2024-06-13 00:00 +0800 by ZhangJie ‐ 2 min read

分享:  

git diff 的默认输出说实话不太好看。改了几行代码，跑个 git diff，满屏的 + -，行内到底改了什么字，得瞪大眼睛去找。IDE 里 side-by-side 对比确实直观，但命令行用户不想为了看个 diff 就切到 GUI 工具——技能应该可迁移，不能被某个平台或 IDE 绑死。

delta 就是解决这个问题的。它是一个 Rust 写的二进制程序，接受 git diff 的输出，然后用更可读的方式渲染出来：side-by-side 对比、word 级别差异高亮、语法高亮、行号等等。

安装 ##

各平台都有包：

# macOS
brew install git-delta

# Debian / Ubuntu
apt-get install git-delta

# Fedora
yum install git-delta

# 如果发行版比较老，仓库里没有，用 cargo 装
cargo install git-delta --locked

装完之后二进制叫 delta，确认一下：

$ delta --version
delta 0.17.0

配置为 Git Pager ##

delta 最常用的姿势是替代默认的 less 作为 git 的 pager。在 ~/.gitconfig 里加几行：

[core]
    pager = delta

[interactive]
    diffFilter = delta --color-only

[delta]
    navigate = true
    side-by-side = true
    line-numbers = true
  • core.pager = delta：让 git diff、git show、git log -p 等命令的输出经过 delta 渲染。
  • interactive.diffFilter：让 git add -p 等交互式命令也能用 delta 的高亮。
  • delta.navigate：开启后可以用 n / N 在 diff 块之间跳转。

配完之后跑个 git diff，效果立竿见影。

关键特性 ##

Side-by-Side 对比 ##

这是 delta 最直观的功能。在 ~/.gitconfig 里设置：

[delta]
    side-by-side = true

旧版在左边，新版在右边，中间用 │ 分隔，改动一目了然。终端宽度不够时会自动切回普通的上下模式。

Word 级别差异高亮 ##

默认的 git diff 只能标到行级别——整行标绿或标红。delta 可以在行内进一步高亮具体哪些字符变了：

[delta]
    features = line-numbers decorations
    minus-style = syntax "#220000"
    minus-emph-style = syntax "#400000"
    plus-style = syntax "#002200"
    plus-emph-style = syntax "#004000"

minus-emph-style 控制被删除行内具体变动的部分，plus-emph-style 对应新增行内的改动。这在变量名重构、字符串微调时特别有用——不用一行行去对了。

语法高亮 ##

delta 内置了语法高亮，会根据 diff 文件的语言自动上色。大部分情况下不需要额外配置，开箱即用。如果想调主题：

[delta]
    syntax-theme = Monokai Extended

主题列表可以用 delta --list-syntax-themes 查看。

行号 ##

[delta]
    line-numbers = true

会在左侧显示原文件和新文件的行号，方便定位。

一套推荐的配置 ##

[core]
    pager = delta

[interactive]
    diffFilter = delta --color-only

[delta]
    navigate = true
    side-by-side = true
    line-numbers = true
    hyperlinks = true
    hyperlinks-file-link-format = "vscode://file/{path}:{line}"

[merge]
    conflictstyle = diff3

[diff]
    colorMoved = default

hyperlinks 开了之后，点击文件路径就能直接跳到编辑器里对应的位置——配合 VS Code 或支持 file link 的终端体验很好。

merge.conflictstyle = diff3 让冲突时 showing 三方对比，delta 会分别渲染。

总结 ##

delta 解决的问题很单一：让 git diff 在终端里好看、好读。它不改变 git 的行为，只是在输出上加了一层渲染。对于常年在命令行里干活的人来说，花五分钟配好就能一直用，值得。

配置完成后跑 git diff 和 git log -p 看看效果。如果终端颜色不对劲，可能是终端主题和 delta 的 syntax-theme 冲突了，换个主题就行。
