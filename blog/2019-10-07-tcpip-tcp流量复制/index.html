TCP 流量复制转发：tcpcopy + intercept

Posted 2018-10-07 00:00 +0800 by ZhangJie ‐ 1 min read

分享:  

背景 #

在服务开发和测试过程中，经常需要用真实线上流量来验证新版本的正确性和性能。tcpcopy 是一个开源的 TCP 流量复制工具，可以将生产环境的 TCP 流量实时复制到测试环境，进行流量回放测试。

正文 #

基本用法 ##

客户端（流量发起端） ##

在需要复制流量的机器上运行 tcpcopy，将本地 15251 端口的流量向远程 10.213.144.140:18108 转发：

./tcpcopy -x 15251-10.213.144.140:18108 -s 10.213.144.140 -c 99.99.99.x -d
  • -x：指定源端口和目标地址端口映射
  • -s：指定 intercept 服务端地址
  • -c：指定伪装客户端的 IP 地址
  • -d：以 daemon 方式运行

服务端（流量接收端） ##

在目标测试服务器上运行 intercept，监听 18108 端口：

./intercept -i eth1 -F "tcp and src port 18108" -d
  • -i：指定网络接口
  • -F：指定 BPF 过滤规则
  • -d：以 daemon 方式运行

工作原理 ##

tcpcopy 工作在网络层，它会：

  1. 捕获指定端口上的 TCP 数据包
  2. 修改目标 IP 和端口，将包发送到测试服务器
  3. intercept 在测试服务器上拦截返回的响应包，避免干扰真实客户端

这样测试服务器就能接收到与生产环境完全相同的请求流量。

总结 #

tcpcopy + intercept 组合是进行线上流量回放测试的有效工具，特别适合在无法直接镜像网络流量的场景下使用。使用时需要注意：

  • 确保测试环境的数据隔离，避免污染线上数据
  • 注意 intercept 端需要能捕获到测试服务器的响应流量
  • 大规模流量复制时注意网络带宽消耗
