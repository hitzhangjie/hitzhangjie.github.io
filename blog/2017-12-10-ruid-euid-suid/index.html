理解 Linux 进程的三种 UID：Real、Effective、Saved Set-UID

Posted 2017-12-10 00:00 +0800 by ZhangJie ‐ 2 min read

分享:  

背景 #

Linux 的权限模型中，每个进程都拥有三种用户 ID：real uid、effective uid 和 saved set-user-id。它们是 Linux 权限控制和特权管理（特别是 setuid 程序）的基石。

本文基于 APUE 中的总结，配合一个 C 语言示例来帮助理解这三者的关系。

正文 #

三种 UID 的定义 ##

UID 类型含义
Real UID (ruid)实际用户 ID，标识"谁启动了这个进程"。通常是登录用户或父进程的 ruid
Effective UID (euid)有效用户 ID，用于权限检查。内核在判断进程能否能执行某个操作（如读写文件）时看的是这个值
Saved Set-UID (suid)保存的设置用户 ID，由exec 从 euid 复制而来。用于让进程在临时切换权限后能恢复回来

setuid 位的作用 ##

可执行文件上可以设置 “set-user-ID” 位（chmod u+s）。当这样的程序被执行时：

  • 进程的 effective uid 会被设置为文件属主的 uid
  • 进程的 saved set-uid 会被从 euid 复制

这允许普通用户临时获得文件属主（通常是 root）的权限来执行特定操作。

代码示例 ##

下面通过一个完整的示例来观察三种 UID 的变化：

// main.c
#include <stdio.h>
#include <unistd.h>

int main(int argc, char *argv[])
{
    uid_t uid, euid, suid;

    getresuid(&uid, &euid, &suid);
    printf("%d %d %d\n", uid, euid, suid);

    setuid(1000);
    getresuid(&uid, &euid, &suid);
    printf("%d %d %d\n", uid, euid, suid);

    return 0;
}

编译并设置 setuid 位：

$ gcc -o main main.c
$ sudo chown lightdm:lightdm main    # lightdm 的 uid 为 108
$ sudo chmod u+s main                 # 设置 set-user-ID 位
$ whoami
zhangjie                              # uid 为 1000
$ ./main
1000 108 108
1000 1000 108

输出分析 ##

第一行 1000 108 108：

  • ruid = 1000：进程由用户 zhangjie（uid=1000）启动，ruid 不会改变
  • euid = 108：因为程序文件属主是 lightdm（uid=108）且设置了 setuid 位，exec 时将 euid 设为 108
  • suid = 108：exec 时从 euid 复制

第二行 1000 1000 108：

  • 程序中调用 setuid(1000)，对于普通用户来说，setuid 只能将 euid 设置为 ruid（1000）或 suid（108）
  • 这里将 euid 设回了 ruid（1000），即放弃特权
  • 如果之后需要恢复特权，可以再 setuid(108)（即 suid 的值）

为什么需要 saved set-uid？ ##

考虑这样的场景：一个 setuid 程序在执行过程中需要临时降低权限（euid → ruid），之后再恢复。如果没有 saved set-uid，一旦放弃特权，就永远无法恢复了。

saved set-uid 的意义在于：保存 exec 时的 euid，供进程之后恢复特权使用。

注意：你可能会想，既然 saved set-uid 存的是特权 uid，我后续可以用来恢复特权，那我在自己的普通程序中直接 setuid(0) 不就能提权了吗？答案是不能。

内核在 setuid() 系统调用中会检查当前进程实际的 saved set-uid（也就是 getresuid() 返回的那个 suid）。一个普通进程中，saved set-uid 就是当前用户的 uid（比如 1000），跟你传入的参数对不上，内核就会拒绝。只有真正从 setuid 可执行文件启动的进程，它的 saved set-uid 才会是特权值，才有资格切换。

总结 #

  • Real UID：谁启动了进程 → 不会变
  • Effective UID：权限检查用谁的身份 → exec setuid 程序时改变
  • Saved Set-UID：特权的"存档" → 从 exec 时的 euid 复制，用于恢复权限
  • setuid() 允许将 euid 设置为 ruid 或 suid，这是临时降低/恢复权限的基础
