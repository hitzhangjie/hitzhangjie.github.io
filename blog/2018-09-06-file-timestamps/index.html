Linux 文件时间戳：atime、ctime、mtime

Posted 2018-09-06 00:00 +0800 by ZhangJie ‐ 1 min read

分享:  

背景 #

Linux 文件系统中的每个文件和目录都维护了三个时间戳，分别记录了不同类型的操作时间。理解它们的区别对于文件管理、备份策略和系统调试都很重要。

正文 #

三个时间戳 ##

缩写全称含义更新时机
atimeAccess Time访问时间文件被读取（read、cat 等）时更新
ctimeChange Timeinode 变更时间文件元数据（权限、所有者、链接数等）变更时更新
mtimeModify Time内容修改时间文件内容被写入修改时更新

关键区别 ##

ctime vs mtime ##

最容易混淆的是 ctime 和 mtime：

  • mtime 只在文件内容变化时更新。例如用 echo "hello" > file 修改了文件内容。
  • ctime 在文件 inode 元数据变化时更新。修改权限（chmod）、修改所有者（chown）、修改硬链接数都会触发 ctime 更新。

注意：ctime 不是"创建时间"（Creation Time）！Linux 传统文件系统没有记录文件创建时间。ctime 中的 “c” 是 “change”，指 inode 变更。

修改内容也会触发 ctime ##

当文件内容被修改时，mtime 会更新，同时 ctime 一般也会被更新（因为文件大小等 inode 信息可能改变）。但反过来，修改权限只更新 ctime，不影响 mtime。

查看文件时间戳 ##

使用 stat 命令可以一次性查看所有三个时间戳：

$ stat filename
  File: filename
  Size: 1234       Blocks: 8          IO Block: 4096   regular file
Access: 2024-01-15 10:30:00.000000000 +0800
Modify: 2024-01-14 15:20:00.000000000 +0800
Change: 2024-01-14 15:20:00.000000000 +0800

使用 ls 配合参数也可以：

  • ls -l：显示 mtime
  • ls -lu：显示 atime
  • ls -lc：显示 ctime

atime 的性能影响与优化 ##

每次文件读取都更新 atime 会产生大量磁盘 I/O。现代 Linux 可以通过 mount 选项优化：

  • relatime（默认）：只有当 atime 早于 mtime 或 ctime 时才更新
  • noatime：完全不更新 atime，适合高 I/O 场景
  • strictatime：严格更新 atime（传统行为）

总结 #

  • atime：读文件时更新（可用 relatime/noatime 优化）
  • mtime：文件内容变化时更新
  • ctime：inode 元数据变化时更新，不是创建时间
