LLM 训练全景：从 GPT-1 到 o1 的进化与对齐

Posted 2025-12-31 00:00 +0800 by ZhangJie ‐ 4 min read

分享:  

背景 ##

Base model 为什么「知道很多事」却不太会聊天？SFT 之后为什么还要再对齐？LoRA 又是在动哪些参数？

这篇文章按一条主线串：能力主要在预训练，行为与偏好在后训练；中间夹参数高效微调，最后补 2024–2025 的偏好优化演进和 o1「更会想」又是如何去做的。

一、规模与范式：变大的不只是参数 ##

先看 GPT 家族七年里，方法论到底换了什么。表里后半段数字多为业界估算或未证实流传，只作数量级对照，不是官方规格。

模型年份参数量硬件 / 成本（口径）范式上真正变了什么
GPT-12018117M（公开）约 8 GPU 量级；成本数千美元级先大规模预训练，再任务微调
GPT-220191.5B（公开）中等规模加速器；约数万美元级（估算）出现可观察的 zero-shot 行为
GPT-32020175B（公开）约千卡 V100 量级（常见估算）；云价等价约数百万美元in-context few-shot；「烧算力」进入主流叙事
GPT-42023未官方披露；流传约万亿级稀疏 MoE训练成本公开表述过「亿美元级」；卡数多为流传多模态 + 更强通用能力；架构细节未证实
GPT-4o2024未公开单次 run 成本未公开（勿与集群 CapEx 混谈）产品向：延迟、多模态、一体化体验
o12024未公开同上后训练 RL + 测试时计算，优化「会推理」

几个转折（只保留对训练有用的） ##

GPT-1：证明「大规模无监督预训练 + 小数据微调」可行。

GPT-2 → GPT-3：规模上去后，zero-shot 苗头变成 few-shot / in-context learning 的产品叙事。所谓「涌现能力」一度被写成「跨过某个参数阈值就突然学会」；后续有工作指出，不少阶跃来自离散评测指标，换成连续指标后往往更平滑。对工程更有用的是：看 scaling 曲线和指标怎么选，别神话阈值。

GPT-4：能力跃迁清楚，架构公开信息极少。业界广泛猜测为 稀疏 MoE（总参大、每 token 只激活部分专家），但 OpenAI 从未官方确认「1.8T」一类数字。MoE 也不是 GPT-4 发明的（GShard / Switch 等更早）。

算力账不只看参数：Chinchilla（2022）强调参数量与训练 token 数要联合缩放；当代很多模型在固定算力下反而 堆更多 token（重复数据、多 epoch），因为推理侧更在意单位算力效果。预训练成本 ≈ tokens × 有效计算，不是「参数一行」能概括的。

o1：不是「再大一号 GPT-4」，而是把 scaling 轴从纯 pretrain 扩到 train-time RL 和 test-time compute（想更久、更贵，换难题表现）。下文单独一节。

二、训练管线：从 Base 到「会听话」 ##

ChatGPT 一类助手不是一次训完，而是分阶段、换目标：

随机初始化
       ↓  海量无标签文本 · 自监督 next-token
Base Model（会续写，不会稳定聊天）
       ↓  指令 / 对话示范 · SFT
SFT 模型（会按格式回答，仍可能阿谀、拒答不当、目标错位）
       ↓  同一 prompt 多回答 · 人类（或 AI）偏好标注
Reward Model（把「哪个更好」拟合成可打分的代理）
       ↓  on-policy RL（经典路径：PPO）· 奖励 ≈ r_RM − β·KL(π ‖ π_ref)
对齐后模型

ps: 实现上 SFT / 偏好优化都可以 全参或 LoRA——那是「更新哪些参数」，不是管线多出来的第 5 阶段。见第三节。

阶段 1：预训练（Pre-training） ##

在网页、书籍、代码等文本上做 next token prediction。学到的是语料里的统计结构：语言规律、世界知识的近似、一定的模式外推能力——不是「完全学会常识和可靠推理」。产物是 Base Model：适合续写和补全，不会稳定遵循指令、控制格式或遵守安全策略。

数据配比（web / code / math / 多语）、去重与质量过滤、context 长度，都会塑形成能力形状。 tokenizer（BPE 等）决定压缩率和有效上下文，工程上躲不掉。

阶段 2：监督微调（SFT） ##

用 Prompt + 理想回答（指令集、多轮对话、工具调用轨迹等）继续训 Base。目标从「续写最可能的下一词」偏到「按示范分布回答」。数据量相对预训练很小，质量要求高；来源也不再只有人工手写——合成数据、模型蒸馏、公开指令集都很常见。

SFT 之后模型「会聊天」了，但仍常有：表面模仿话术、有害请求处理不当、过度迎合、以及 next-token 似然 ≠ 人类想要的回答分布 带来的错位。所以还要偏好对齐。

阶段 3：奖励模型（RM） ##

经典 RLHF 的关键一步：

  1. 用 SFT 模型对同一 Prompt 采多个回答（常见 4–9 个）
  2. 标注员做 成对比较（pairwise preference；若给了全排序，通常会展开成多对）
  3. 用 Bradley-Terry 一类假设训 Reward Model：输入 (prompt, response)，输出标量分数

RM 是人类偏好的可扩展代理，不是「绝对真理打分器」。它会继承标注偏差，也容易被后来的策略 刷分（reward hacking：冗长、阿谀、空洞但高分）。

阶段 4：强化学习优化（经典路径：PPO） ##

把 SFT 模型当作初始策略 (\pi_\theta)，再挂四件套：

角色是什么干什么
Policy正在更新的 LLM按 prompt 生成回答
Reference (\pi_{\text{ref}})通常冻结的 SFT 副本KL 的参照，防止策略飘太远
Reward Model偏好打分器给整段(x,y) 标量奖励（多为序列级）
Value / Critic常共享 backbone 的 value head估(V(s))、算 advantage，不替代 RM

最终送给优化器的奖励大致是：

[ r = r_{\mathrm{RM}}(x,y) - \beta,\mathrm{KL}\big(\pi_\theta(\cdot|x),|,\pi_{\mathrm{ref}}(\cdot|x)\big) ]

实现上 KL 多对采样轨迹做序列 log-ratio 求和（对 (y\sim\pi_\theta) 的近似），写进 shaped reward，而不是「PPO 之外再随手加一项」。目的包括：缓解 reward hacking、保持流畅与不塌缩到刷分话术。

循环直觉：policy 采样 → RM（−KL）给奖励 → value 估优势 → PPO 更新 policy（并更新 value）。动作是 逐 token，RM 奖励却常是 整段一个分——所以需要 advantage / baseline 做信用分配。

一句话：RLHF = 用偏好数据训打分器，再让策略在「高分且别离 ref 太远」的约束下优化。

工业上还有 Best-of-N / rejection sampling（多采样再按 RM 挑）等更轻的手段，不一定上满血 PPO。

三、参数怎么更新：全参、LoRA 与 QLoRA ##

上一节回答「用什么数据、优化什么目标」。这一节是横切问题：更新哪些参数。SFT、领域适配、甚至不少 DPO 训练，都可以选全参或 PEFT。

全参数微调 ##

所有权重参与更新。数据够、资源够时上限往往最高；小数据上更容易过拟合和灾难性遗忘。显存不只是「多存一份梯度」——常见还有 优化器状态（Adam 约 2× 参数量） 和 激活，粗算远超「模型大小 + 一份梯度」。

LoRA（Low-Rank Adaptation） ##

冻结原权重 (W)，旁路两个低秩矩阵：

h = W·x + (α/r)·B·A·x
    （W 冻结；只训练 A、B；α 为 lora_alpha，r 为 rank）

(A \in \mathbb{R}^{r \times d_{\mathrm{in}}})，(B \in \mathbb{R}^{d_{\mathrm{out}} \times r})。可训练参数量随 r 和 挂载模块 变：只挂 Q/V、r=8 时可能到千分之一量级；all-linear + 较大 r 会到百分级。

为什么够用：任务适配对应的 (\Delta W) 常常近似低秩，用 (BA) 参数化就能逼近有效更新。

推理前可把 (BA) merge 进 (W)，单 adapter 时不增加推理算子；多任务多 adapter 动态切换时通常不能 merge 成一份 W。常见挂载从论文的 Q/V 到现在的 q/k/v/o + MLP 都有。

QLoRA ##

在 4bit（如 NF4）量化的冻结基座上再挂 LoRA：LoRA 省的是 可训练参数与对应优化器；QLoRA 再省 基座权重显存。单卡 SFT 7B/13B 级，这条路径基本是默认技能点。

Prompt Tuning（谱系边缘） ##

不改 Transformer 权重，只在输入侧学一组 soft prompt 向量。参数极少，但推理必须带着这些向量；和 LoRA「可 merge 进 W」的部署形态不同。Prefix Tuning 是在各层 K/V 前插前缀，不要和入口 Prompt Tuning 混成一种。现代大模型指令微调主力仍是 全参或 LoRA/QLoRA。

方式基座权重额外结构训练对象部署备注
全参更新无全部参数权重即产物
LoRA默认冻结旁路低秩(BA)A、B可 merge
QLoRA量化冻结同上A、B显存友好
Prompt Tuning冻结虚拟 token 向量soft prompt推理需携带

四、PPO 在 RLHF 里具体在挡什么 ##

PPO（Proximal Policy Optimization）是经典 RLHF 里常用的 on-policy 算法。相对前身 TRPO：去掉复杂二阶约束，用一阶优化 + clip。

Clipping 裁的是概率比，不是「策略差」黑盒 ##

设重要性采样比 (r_t(\theta)=\pi_\theta(a_t|s_t)/\pi_{\theta_{\mathrm{old}}}(a_t|s_t))，目标大致是：

[ \min\big(r_t\hat A_t,;\mathrm{clip}(r_t,1-\varepsilon,1+\varepsilon)\hat A_t\big) ]

它在 surrogate 上对 过大的概率比更新 做悲观裁剪，降低一次更新把策略扯崩的风险——不是 TRPO 那种硬 KL 信赖域，也不是梯度裁剪的同义词。

Actor 与 Critic 别和 RM 搞混 ##

  • Actor = policy = LLM
  • Critic = value head，服务 (\hat A)（可用 GAE 等估计）
  • 打分 = RM（再减 KL）

没有 advantage，clip 公式里的 (\hat A_t) 无从谈起；没有 RM，value 也成不了「人类偏好」的来源。

探索在 LLM 里长什么样 ##

不是 ε-贪婪（那是价值方法在离散控制里常用的套路）。LLM-PPO 的探索主要来自 按 (\pi_\theta) 采样（temperature、top-p 等），再用相对 (\pi_{\mathrm{ref}}) 的 KL 把「乱探索 / 刷分」拴住。

相对 DPO 等离线偏好方法：PPO 要 在线采样 + RM +（常有）value，工程更重；大规模、需要持续探索时头部实验室仍常用，更多产线则转向直接偏好优化。

五、对齐在变：从 PPO 全家桶到直接偏好优化 ##

经典「RM → PPO」仍是重要范式，但 不是 2024–2025 唯一主流。动机很具体：少训一个 RM、少跑 on-policy 采样、训练更稳。

方法还要 RM 吗还要 on-policy RL 吗换来什么
PPO-RLHF要要灵活、可在线探索；链路长、易不稳
DPO不显式训不要用偏好对直接优化策略（BT + 最优策略对奖励的 reparameterization）；更省、常更稳
ORPO / KTO 等视变体通常不要进一步简化数据形态或损失
RLAIF / Constitutional AI偏好可来自 AI可仍做 RL降人工标注；CAI 还有「按原则改写 → 再偏好」的流程，不等于裸 LLM-as-Judge
GRPO 等常配合规则/结果奖励要 RL，但可无 value model推理模型训练里常见（组内相对优势）

Best-of-N、rejection sampling、以及「SFT 足够好的领域」也仍然有人用——对齐是工具箱，不是唯一宗教。

失败模式跨方法共通：标注/裁判偏差、长度与风格 bias、reward（或隐式奖励）被钻空子、过度对齐导致能力回退。换算法不会自动等于「对齐到人类价值观」——优化的是 产品定义的偏好分布。

六、o1：另一条轴——更会想，而不只是更大 ##

标题写到 o1，这里把差异说死：

经典 Chat 对齐（GPT-4 助手叙事）o1 类推理模型
主目标指令遵循、风格、安全、有用可验证任务上的推理质量（数学、代码等）
典型信号人类/AI 偏好结果正确性、过程监督等（公开细节有限）
算力轴主要靠 pretrain + 后训练再加test-time compute（生成更长思考轨迹）
用户感知更快、更会聊天更慢、更贵、难题上更稳

OpenAI 公开表述的核心是：用大规模强化学习训练模型使用 chain-of-thought，性能随 训练期算力 和 测试期算力 一起提升。具体是否 ORM/PRM、奖励如何设计，细节未完全公开——写到「优化对象从偏好回答扩到可验证的推理过程 + 愿意在推理时多花算力」就够用，不必把传闻当规格表。

和第五节的关系：推理模型训练往往仍是 RL，但不等于「再跑一遍聊天 RLHF」；规则奖励、组相对方法（如 GRPO）在开源推理模型里更常见。

总结 ##

读完应能画出一张图，并记住两个判断：

  1. 能力底座在预训练（参数 × 数据 × 算力）；「会不会按人的方式回答」主要在 SFT + 偏好对齐。 LoRA/QLoRA 只是怎么更新参数，贯穿后训练，不是另起的阶段。
  2. 经典 RLHF 是 RM +（KL 约束下的）PPO；大量场景已用 DPO 族绕开 RM 与 on-policy。 o1 把故事从「更听话的助手」扩到「更会想的推理系统」——pretrain 没被取代，后训练和测试时计算成了新轴。

从 GPT-1 到 o1，美元与卡数在涨，但主线一直是：先学会语言与知识的统计结构，再学会按目标分布说话和推理。
