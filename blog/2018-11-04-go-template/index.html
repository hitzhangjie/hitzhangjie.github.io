Go template 使用详解

Posted 2018-11-04 14:32 +0800 by ZhangJie ‐ 8 min read

分享:  

背景 #

模板技术在软件开发中有着广泛的应用场景：Web 页面渲染、邮件内容生成、自动化代码生成、CLI 输出定制等等。Go 语言在标准库中内置了强大的模板引擎 text/template（以及安全的 HTML 版本 html/template），无需引入第三方库即可完成模板解析与执行。

本文先梳理 Go 模板编程的核心概念与基础语法，建立知识框架，再通过层层递进的示例展示实际用法，最后给出一个代码生成实战案例。

核心概念与基础语法 #

在开始写代码之前，先理解 Go 模板的运转模型和几个关键概念。

运转模型 ##

Go 模板的运转过程非常直观：

模板文本 (string)  ──Parse──▶  *Template  ──Execute(data)──▶  输出 (io.Writer)
  • 模板文本：一段包含"动作"（actions）的普通文本，动作用于控制输出内容
  • 解析（Parse）：将模板文本编译为内部表示，得到 *Template 对象
  • 执行（Execute）：将数据绑定到模板，遍历数据结构，把结果写入 io.Writer

模板可以被重复执行——解析一次，用不同数据执行多次，无需重新解析。

动作（Actions）与定界符 ##

模板中所有动态行为都通过动作来表达。动作用 {{ 和 }} 括起来，这两个符号称为定界符（delimiters）。定界符之外的文本会原样输出。

Hello, {{.Name}}!           →  Hello, Alice!
{{.Count}} items in stock.   →  42 items in stock.

动作可以分为两大类：

类别说明示例
数据求值获取数据的值并输出{{.Name}}、{{.Age}}
控制结构控制模板的执行流程{{if}}、{{range}}、{{with}}、{{define}}、{{template}}、{{block}}

Dot（.）—— 当前作用域的"光标" ##

. 是 Go 模板中最重要的概念，称为 dot。它代表当前作用域中正在被操作的数据，可以理解为游走在数据结构上的"光标"。

  • 模板刚执行时，. 指向 Execute 传入的顶层数据
  • 进入 {{range}} 循环后，. 指向当前迭代的元素
  • 进入 {{with}} 块后，. 指向 with 指定的新值
// 假设传入结构体 {Name: "Alice", Age: 25}
{{.Name}}    // 用 . 访问当前作用域下的 Name 字段 → "Alice"
{{.Age}}     // 用 . 访问当前作用域下的 Age 字段 → 25

理解 . 是理解所有后续语法的前提。简单记住：“点就是我当前在看的数据”。

参数（Arguments）—— 动作中数据的写法 ##

动作内部引用数据时，使用以下参数形式：

写法含义示例
.当前 dot 的值{{.}}
.Field访问结构体字段（链式访问 .F1.F2）{{.Name}}
.Key访问 map 键（不需要大写开头）{{.count}}
.Method调用无参方法（返回 1 值或 值+error）{{.String}}
$var已声明的变量{{$name}}
$顶层数据——无论 dot 被嵌套了多少层，$ 始终指向 Execute 传入的原始数据{{$.Title}}
fun arg调用函数{{printf "%s" .Name}}

自动解引用：如果值是 pointer、interface，模板引擎会自动 dereference 到基础类型，无需在模板中手动取址。

管道（Pipeline）—— 链式数据流 ##

管道是 Go 模板中最强大的表达方式之一。与 Unix shell 类似，用 | 将多个命令串联起来，前一命令的输出变成下一命令的最后一个参数：

{{"hello" | printf "%s world" | upper}}
// 第一步: printf "%s world" "hello"    → "hello world"
// 第二步: upper "hello world"          → "HELLO WORLD"

管道中的每个命令（| 分隔的片段）名为 command，command 的格式为：

Argument [.Method] [Argument...]
functionName [Argument...]

即一个 command 可以是「参数 + 方法 + 更多参数」，也可以是「函数名 + 参数」。

控制结构一览 ##

Go 模板提供了类编程语言的控制结构：

动作用途
{{if pipeline}} ... {{end}}条件判断（非空为真）
{{if pipeline}} ... {{else}} ... {{end}}带 else 的条件判断
{{if pipeline}} ... {{else if pipeline}} ... {{end}}else-if 链
{{range pipeline}} ... {{end}}遍历切片/map/channel
{{range pipeline}} ... {{else}} ... {{end}}遍历——零长度时走 else 分支
{{with pipeline}} ... {{end}}切换 dot 到新值并执行
{{with pipeline}} ... {{else}} ... {{end}}带空值处理的 with
{{break}} / {{continue}}跳出/跳过当前 range 循环
{{define "name"}} ... {{end}}定义一个具名子模板
{{template "name" pipeline}}调用具名子模板（pipeline 成为子树中的 dot）
{{block "name" pipeline}} ... {{end}}define + 立即 template 调用的语法糖——定义默认内容并可被覆盖

条件中的"空值"判断标准：false、0、nil、空数组/切片/map/字符串视为空（假），其余为非空（真）。

变量 ##

模板中可以用 $varname 声明和使用变量：

{{$name := .FirstName}}      // 声明并赋值
{{$name = .LastName}}         // 重新赋值（注意用 = 而非 :=）
{{range $idx, $elem := .Items}} ... {{end}}   // range 中声明索引和元素变量

变量作用域从声明处到所在控制结构的 {{end}}，模板调用不继承变量。

注释 ##

{{/* 这是模板注释，不会出现在输出中 */}}

注释可以跨行，但不能嵌套。

FuncMap —— 注册自定义函数 ##

模板默认只提供一组内置函数（比较、逻辑、格式化等）。要扩展功能，需要通过 FuncMap 注册自定义函数：

funcMap := template.FuncMap{
    "funcName": yourFunc,
}
tpl := template.New("name").Funcs(funcMap).Parse(tplText)

Funcs() 必须在 Parse() 之前调用，否则解析模板时找不到自定义函数。

每个注册的函数必须返回 (value) 或 (value, error)。如果 error 非 nil，模板执行会终止。

template.Must —— 初始化时强制校验 ##

template.Must 包装了解析结果：如果 parse 出错就 panic，适合在 var 初始化时使用，确保模板在程序启动阶段就是有效的。

var tpl = template.Must(template.New("demo").Parse("Hello, {{.}}"))

以上就是 Go 模板编程的核心知识骨架。下面通过实际代码来一一演示。

基础示例 #

从一个学生成绩汇总的例子开始，演示模板解析、自定义函数、管道操作与数据绑定。

package main

import (
	"fmt"
	"os"
	"text/template"
)

// 自定义函数：根据分数评定等级
// {{.Grade | studentLevel}} 中 | 将 .Grade 作为参数传给 studentLevel
func studentLevel(grade int) string {
	switch {
	case grade > 90:
		return "A"
	case grade > 80:
		return "B"
	case grade > 70:
		return "C"
	case grade > 60:
		return "D"
	default:
		return "E"
	}
}

func main() {
	tplDef := `
{{.Count}} students:
{{range .Students}}
-------------------
Name : {{.Name}}
Age  : {{.Age}}
Grade: {{.Grade}}
Level: {{.Grade | studentLevel}}
{{end}}
`

	// 先注册自定义函数，再解析模板
	tpl, err := template.New("students-summary").
		Funcs(template.FuncMap{"studentLevel": studentLevel}).
		Parse(tplDef)
	if err != nil {
		fmt.Printf("parse template error: %v\n", err)
		os.Exit(1)
	}

	type Student struct {
		Name  string
		Age   int
		Grade int
	}
	type Students struct {
		Count    int
		Students []Student
	}

	book := Students{
		Students: []Student{
			{"zhang", 10, 99},
			{"wang", 10, 80},
			{"li", 10, 70},
		},
	}
	book.Count = len(book.Students)

	if err := tpl.Execute(os.Stdout, book); err != nil {
		fmt.Printf("execute template error: %v\n", err)
	}
}

程序输出：

3 students:
-------------------
Name : zhang
Age  : 10
Grade: 99
Level: A
-------------------
Name : wang
Age  : 10
Grade: 80
Level: C
-------------------
Name : li
Age  : 10
Grade: 70
Level: D

回顾前面提到的概念在这个例子中如何体现：

  • dot：{{.Count}} 访问顶层的 Count 字段；进入 {{range .Students}} 后，. 切换为当前迭代的 Student
  • 管道：{{.Grade | studentLevel}} 中，.Grade 的值通过管道传给 studentLevel 函数
  • Funcs：在 Parse 之前注册，模板中才能直接使用函数名
  • Execute：将数据 book 绑定到模板，输出到 os.Stdout

活用控制结构 #

条件判断 ##

if 支持 else 和 else if 链，条件为空值则为假：

tplDef := `
{{if .Score}}
  {{if eq .Score 100}}
    Perfect!
  {{else if ge .Score 60}}
    Pass
  {{else}}
    Fail
  {{end}}
{{else}}
  No score recorded.
{{end}}
`

这里用到了内置比较函数 eq（equal）和 ge（greater or equal）。完整的内置比较函数为 eq、ne、lt、le、gt、ge。它们支持多参数：eq a b c 等价于 a==b || a==c。

另有逻辑函数 and、or、not，均支持短路求值。

range 进阶 ##

range 遍历数组、切片、map 或 channel：

tplDef := `
{{range $idx, $item := .Items}}
  #{{$idx}}: {{$item.Name}} — {{$item.Value}}
{{else}}
  (empty list)
{{end}}
`
  • $idx 获取索引（数组/切片）或键（map），$item 获取元素值
  • 注意：只声明一个变量时（range $x := ...），$x 接收的是元素值而非索引，这与 Go 语言 for range 的习惯不同
  • {{else}} 分支在遍历对象长度为 0 时执行

with 语句 ##

with 将 dot 切换到新值，简化深层字段的访问：

tplDef := `
{{with .Address}}
  City: {{.City}}
  Street: {{.Street}}
{{else}}
  No address provided.
{{end}}
`

进入 with 后，{{.City}} 等价于原 dot 下 .Address.City。如果 Address 为空值，走 else 分支，dot 不受影响。

break / continue ##

在循环中可按条件提前跳出或跳过：

tplDef := `
{{range .Items}}
  {{if eq .Name ""}}{{continue}}{{end}}
  {{if gt .Value 100}}{{break}}{{end}}
  {{.Name}}: {{.Value}}
{{end}}
`

管道深入与自定义函数 #

管道 | 是 Go 模板的灵魂。串联多个函数，数据逐级流转：

funcMap := template.FuncMap{
	"upper": strings.ToUpper,
	"trim":  strings.TrimSpace,
}

tpl := template.Must(template.New("pipe").Funcs(funcMap).Parse(
	`{{.Name | trim | upper}}`,
))
tpl.Execute(os.Stdout, map[string]string{"Name": "  hello  "})
// 输出: HELLO

每通过一个 |，左侧的结果被追加为下一个函数的最后一个参数。所以 "hello" | printf "%s world" 等价于 printf "%s world" "hello"。

自定义函数签名要求：

  • func(args...) T —— 一个返回值
  • func(args...) (T, error) —— 返回值和 error。error 非 nil 时模板执行终止并返回该 error

在代码生成场景中，常用的自定义函数包括：类型映射（string → string，int32 → int32）、命名风格转换（snake_case → CamelCase）等。

模板嵌套与复用 #

现实场景中，模板往往需要分层、可复用。Go 模板通过关联模板（associated templates）机制支持这一点。

define / template ##

在同一个模板文本中用 {{define}} 定义具名子模板，用 {{template}} 调用：

const layout = `
Header
{{template "body" .}}
Footer
`

const bodyDef = `
{{define "body"}}
  Content for: {{.}}
{{end}}
`

tpl := template.Must(template.New("layout").Parse(layout))
tpl = template.Must(tpl.Parse(bodyDef))  // 将 body 模板关联到同一个 Template 实例
tpl.Execute(os.Stdout, "MyPage")

输出：

Header
  Content for: MyPage
Footer

关键点：

  • {{template "body" .}} 中的 . 是传给子模板的数据（这里将当前 dot 传下去）
  • 可以传 nil，也可以传具体值
  • 子模板必须在同一个 *Template 实例中定义（通过多次 Parse 累积，或通过 ParseFiles/ParseGlob 加载）

block —— 可覆盖的默认内容 ##

block 是 define + 立即 template 调用的语法糖，最适合"提供默认实现，允许覆盖"的场景：

const master = `Names:{{block "list" .}}{{"\n"}}{{range .}}{{println "-" .}}{{end}}{{end}}`

// 场景一：使用默认的 block 定义（逐行列出）
tpl1 := template.Must(template.New("master").Parse(master))
tpl1.Execute(os.Stdout, []string{"Alice", "Bob"})
// Names:
// - Alice
// - Bob

// 场景二：覆盖 list 模板（用逗号连接），需要注册 join 函数
tpl2 := template.Must(tpl1.Clone())
tpl2.Funcs(template.FuncMap{"join": func(a []string, sep string) string {
	return strings.Join(a, sep)
}}).Parse(`{{define "list"}} {{join . ", "}}{{end}}`)
// 输出: Names: Alice, Bob

block 等价于：先 define，如果该名字没有被外部定义过，就用这段默认内容；如果后来用同一个名字再 define 一次，就会覆盖它。

ParseFiles / ParseGlob ##

生产环境中，模板通常放在 .tmpl 文件中：

// 加载单个文件（模板名 = 文件 base name）
tpl := template.Must(template.ParseFiles("layout.tmpl"))

// 加载所有匹配的文件
tpl := template.Must(template.ParseGlob("templates/*.tmpl"))

// 指定模板名
tpl := template.Must(template.New("myname").ParseGlob("templates/*.tmpl"))

ParseFiles/ParseGlob 会将文件中的模板定义（{{define "..."}}）全部加载到同一个 Template 实例下，形成关联模板的命名空间。

实用技巧 #

空白控制 ##

{{- 裁掉前方空白，-}} 裁掉后方空白：

tplDef := `{{23 -}} < {{- 45}}`
tpl := template.Must(template.New("ws").Parse(tplDef))
tpl.Execute(os.Stdout, nil)
// 输出: 23<45

这在用模板生成代码时尤其有用——避免输出文件中出现多余的空行和缩进错乱。

注意 {{-3}} 会被解析为数字 -3，空白控制必须写成 {{- 3}}（中间有空格）。

missingkey 选项 ##

控制访问 map 中不存在的 key 时的行为：

tpl := template.Must(template.New("test").
    Option("missingkey=error").  // default | zero | error
    Parse("{{.Name}}"))
选项行为
missingkey=default（默认）输出 <no value>，静默继续
missingkey=zero返回该类型的零值
missingkey=error立即终止执行并返回 error

建议在代码生成场景中始终使用 missingkey=error，避免因为拼写错误漏过模板字段。

Must —— 初始化校验 ##

var tpl = template.Must(template.New("demo").Parse("Hello, {{.}}"))

Must 在 parse 失败时 panic，适合在包级 var 初始化时使用——程序启动即发现问题，而非运行时出错。

代码生成实战 #

Go 模板在代码生成场景中极具价值。下面用一个简化版的 protobuf + Go 模板生成服务代码的示例，展示完整的代码生成流水线。

数据模型 ##

首先定义描述服务的数据结构：

type ServiceDesc struct {
	Package  string
	Name     string
	HasHTTP  bool   // 内部使用，驱动条件渲染
	Methods  []MethodDesc
}

type MethodDesc struct {
	Name        string
	InputType   string
	OutputType  string
	HTTPMethod  string
	HTTPPath    string
}

自定义函数 ##

var funcMap = template.FuncMap{
	"lower":     strings.ToLower,
	"toCamel":   toCamelCase,
	"goType":    toGoType,
}

func toCamelCase(s string) string {
	parts := strings.Split(s, "_")
	for i, p := range parts {
		parts[i] = strings.Title(p)
	}
	return strings.Join(parts, "")
}

func toGoType(pt string) string {
	mapping := map[string]string{
		"string": "string", "int32": "int32", "int64": "int64",
		"bool": "bool", "float64": "float64", "bytes": "[]byte",
	}
	if t, ok := mapping[pt]; ok {
		return t
	}
	return pt
}

模板定义 ##

Service 接口及 Stub ##

var serviceTmpl = `
// Code generated by protoc-gen-demo. DO NOT EDIT.
// source: {{.Name}}.proto
package {{.Package}}

import (
	"context"
	{{if .HasHTTP}}"net/http"{{end}}
)

// {{.Name}}Service 定义服务接口
type {{.Name}}Service interface {
{{range .Methods}}
	// {{.Name}}
	{{.Name}}(ctx context.Context, req *{{.InputType | goType}}) (*{{.OutputType | goType}}, error)
{{end}}
}

// {{.Name}}ServiceStub 服务端骨架
type {{.Name}}ServiceStub struct {
	impl {{.Name}}Service
}

func New{{.Name}}ServiceStub(impl {{.Name}}Service) *{{.Name}}ServiceStub {
	return &{{.Name}}ServiceStub{impl: impl}
}

{{range .Methods}}
func (s *{{$.Name}}ServiceStub) Handle{{.Name}}(w http.ResponseWriter, r *http.Request) {
	// TODO: 反序列化请求 → s.impl.{{.Name}}(ctx, req) → 序列化响应
}
{{end}}
`

注意 {{$.Name}} —— 在 range 内部，. 变成了当前 Method，要访问父级的 Name 字段，就用 $ 回到顶层数据。

Client 代理 ##

var clientTmpl = `
package {{.Package}}

import "context"

// {{.Name}}Client 客户端代理
type {{.Name}}Client struct {
	// httpClient *http.Client
	// baseURL    string
}

func New{{.Name}}Client(addr string) *{{.Name}}Client {
	return &{{.Name}}Client{}
}

{{range .Methods}}
func (c *{{$.Name}}Client) {{.Name}}(ctx context.Context, req *{{.InputType | goType}}) (*{{.OutputType | goType}}, error) {
	// TODO: 序列化请求 → HTTP 调用 → 反序列化响应
	return nil, nil
}
{{end}}
`

生成与格式化 ##

func generate(svc ServiceDesc) error {
	svc.HasHTTP = true

	// 生成 service 代码
	var buf bytes.Buffer
	tpl := template.Must(template.New("service").Funcs(funcMap).Parse(serviceTmpl))
	if err := tpl.Execute(&buf, svc); err != nil {
		return fmt.Errorf("execute service template: %w", err)
	}
	// 用 go/format 格式化生成代码
	formatted, err := format.Source(buf.Bytes())
	if err != nil {
		// 格式化失败时仍保留原始文件供排查
		fmt.Printf("format warning: %v\n", err)
		formatted = buf.Bytes()
	}
	if err := os.WriteFile(
		fmt.Sprintf("%s_service.go", strings.ToLower(svc.Name)), formatted, 0644,
	); err != nil {
		return err
	}

	// 生成 client 代码
	buf.Reset()
	tpl2 := template.Must(template.New("client").Funcs(funcMap).Parse(clientTmpl))
	if err := tpl2.Execute(&buf, svc); err != nil {
		return fmt.Errorf("execute client template: %w", err)
	}
	formatted, err = format.Source(buf.Bytes())
	if err != nil {
		fmt.Printf("format warning: %v\n", err)
		formatted = buf.Bytes()
	}
	return os.WriteFile(
		fmt.Sprintf("%s_client.go", strings.ToLower(svc.Name)), formatted, 0644,
	)
}

格式化方式对比 ##

除了 go/format.Source()，实际项目中还有两种常用方式：

import (
	"bytes"
	"fmt"
	"go/format"
	"os/exec"
)

// 方式一：go/format.Source —— 纯 Go，不依赖外部命令，适合 library
func formatSource(src []byte) ([]byte, error) {
	return format.Source(src)
}

// 方式二：goimports —— 同时格式化 import 和代码，适合 CLI 工具
func formatWithGoimports(src []byte) ([]byte, error) {
	cmd := exec.Command("goimports")
	cmd.Stdin = bytes.NewReader(src)
	var out bytes.Buffer
	cmd.Stdout = &out
	cmd.Stderr = &out
	if err := cmd.Run(); err != nil {
		return nil, fmt.Errorf("goimports: %w\n%s", err, out.String())
	}
	return out.Bytes(), nil
}

// 方式三：gofmt —— 只格式化代码（不处理 import），适合简单场景
func formatWithGofmt(src []byte) ([]byte, error) {
	cmd := exec.Command("gofmt")
	cmd.Stdin = bytes.NewReader(src)
	return cmd.Output()
}

推荐的代码生成流水线：

解析输入 (proto/YAML/DB schema)
  → 构建数据模型 (structs)
  → text/template 渲染 (Execute)
  → format.Source / goimports 格式化
  → 写入目标文件

这样生成的代码不仅语义正确，格式也完全符合 Go 社区规范，可以直接提交到仓库。

text/template 与 html/template #

Go 标准库提供了两套模板包，接口完全一致：

特性text/templatehtml/template
用途通用文本输出HTML 页面渲染
自动转义❌✅（防 XSS）
适用场景代码生成、邮件、CLIWeb 页面

html/template 会根据上下文自动选择转义策略：HTML 标签体按 HTML 转义，<script> 内按 JS 转义，href 属性中按 URL 转义，从而有效防止 XSS 攻击。如果生成的是非 HTML 文本，用 text/template 即可，不必引入额外的转义开销。

总结 #

本文从概念到实战，系统介绍了 Go 模板编程的方方面面：

  • 概念基石：dot（.）是理解一切的关键——它是游走于数据之上的光标；管道 | 让数据在函数间流转，表达力极强
  • 控制结构：if/else、range/else、with/else、break、continue——覆盖常见逻辑控制需求
  • 函数体系：内置的比较/逻辑/格式化函数，加上 FuncMap 注册自定义函数，扩展灵活
  • 模板复用：define/template 定义和调用子模板，block 提供"默认+可覆盖"模式，ParseFiles/ParseGlob 加载外部文件
  • 工程化：Must 初始化校验、空白控制、missingkey 选项、Clone 复制模板集
  • 代码生成：配合 go/format 或 goimports 对生成代码做格式化，产出可直接提交的规范代码

如果你之前只用过 ctemplate、velocity 等外部模板引擎，你会发现 Go 的内置模板同样强大，而且零依赖、与 Go 类型系统天然契合。
