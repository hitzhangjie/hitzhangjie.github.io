操作系统进程状态详解

Posted 2019-07-05 00:00 +0800 by ZhangJie ‐ 1 min read

分享:  

背景 #

进程状态是操作系统进程管理的基础概念。不同教材和不同角度（CPU 视角 vs 虚拟内存视角）会给出不同的状态分类。本文从两个维度完整梳理进程状态及其转换关系。

正文 #

CPU 视角：三种基础状态 ##

从 CPU 的角度来看，它只关心"当前执行谁"，因此只需要三种状态：

状态含义
运行态（Running）进程当前拥有 CPU，正在执行指令
就绪态（Ready）进程拥有所有需要的资源，只等 CPU 调度
阻塞态（Blocked）进程因等待某个事件（如 I/O 完成、锁释放）而暂时放弃 CPU

状态转换：

  • 运行 → 阻塞：进程发起 I/O 操作，主动让出 CPU
  • 运行 → 就绪：时间片用完，被调度器抢占
  • 阻塞 → 就绪：等待的 I/O 事件完成
  • 就绪 → 运行：被调度器选中

虚拟内存视角：引入两个挂起状态 ##

当系统物理内存不足时，操作系统会进行交换（swap），将部分进程的内存映像换出到磁盘交换区。这时就引入了两个额外的状态：

挂起-阻塞态（Suspended-Blocked） ##

进程原本处于阻塞态（正在等待 I/O），同时其内存映像被换出到磁盘：

  • 如果 I/O 事件完成 → 转入挂起-就绪态（仍在外存，等待换入）
  • 如果被换入物理内存 → 转入阻塞态（继续等待 I/O 完成）

挂起-就绪态（Suspended-Ready） ##

进程原本处于就绪态（可以运行），但其内存映像被换出到磁盘：

  • 如果被换入物理内存 → 转入就绪态（可以参与调度）
  • 挂起-阻塞态的进程 I/O 完成后也会转入此状态

         调度选中    │  运行态  │  等待I/O事件

总结 #

  • 3 种 CPU 基础状态：运行、就绪、阻塞
  • 2 种挂起状态：挂起-阻塞、挂起-就绪，由虚拟内存交换（swap）引入
  • 挂起状态是独立于 CPU 调度状态的"第二维度"
  • 理解进程状态对分析系统性能（如负载、I/O 等待）至关重要
