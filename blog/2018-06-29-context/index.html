Go context 包使用指南

Posted 2018-06-29 14:32 +0800 by ZhangJie ‐ 3 min read

分享:  

背景 ##

在 Go 服务端开发中，每个请求通常会跨越多个 goroutine：处理 HTTP 请求、查询数据库、调用下游 RPC 等等。当请求被取消或超时时，我们希望整个调用链上的所有 goroutine 都能及时停止，避免资源浪费。

Go 标准库 context 包正是为此而生。它在 API 边界和进程间传递 deadline、取消信号 以及 请求级别的参数。

context 的设计 ##

context.Context 是一个接口，核心方法有四个：

type Context interface {
    Deadline() (deadline time.Time, ok bool)  // 返回截止时间
    Done() <-chan struct{}                     // 返回一个只读 channel，取消时关闭
    Err() error                                // 返回取消原因
    Value(key interface{}) interface{}         // 获取绑定的值
}

context 包提供了四个创建派生 Context 的函数，覆盖了所有使用场景：

函数用途
context.WithCancel(parent)手动取消
context.WithDeadline(parent, time)在指定时刻自动取消
context.WithTimeout(parent, duration)在指定时长后自动取消
context.WithValue(parent, key, value)携带键值对

所有函数都接收一个父 Context，返回一个派生 Context。多个 Context 可以形成父子链，父 Context 取消时，所有子 Context 都会被级联取消。

WithCancel：主动取消 ##

当需要手动控制 goroutine 的退出时，使用 WithCancel：

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel() // 确保最终取消，防止 goroutine 泄漏

    go worker(ctx)

    time.Sleep(5 * time.Second)
    cancel() // 主动通知 worker 退出

    time.Sleep(1 * time.Second) // 等待 worker 清理
}

func worker(ctx context.Context) {
    for {
        select {
        case <-ctx.Done():
            fmt.Println("worker: received cancel signal, exiting")
            return
        default:
            fmt.Println("worker: doing work...")
            time.Sleep(1 * time.Second)
        }
    }
}

重要：即使没有主动调用 cancel()，也要用 defer cancel() 确保资源释放，防止 goroutine 泄漏。多次调用 cancel() 是安全的（只有第一次生效），所以 defer cancel() 不会有副作用。

WithDeadline：按时刻取消 ##

当需要指定某个具体时间点自动取消时，使用 WithDeadline：

func main() {
    // 2 秒后自动取消
    deadline := time.Now().Add(2 * time.Second)
    ctx, cancel := context.WithDeadline(context.Background(), deadline)
    defer cancel()

    req, _ := http.NewRequestWithContext(ctx, "GET", "https://example.com", nil)
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        fmt.Println("request failed:", err) // 超时会走到这里
        return
    }
    defer resp.Body.Close()
    // 处理响应...
}

WithTimeout：按时长取消 ##

WithTimeout 是 WithDeadline 的便捷封装，底层等价于 WithDeadline(parent, time.Now().Add(timeout))：

func main() {
    // 2 秒后自动取消
    ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
    defer cancel()

    req, _ := http.NewRequestWithContext(ctx, "GET", "https://example.com", nil)
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        fmt.Println("request failed:", err)
        return
    }
    defer resp.Body.Close()
    // 处理响应...
}

实际开发中 WithTimeout 比 WithDeadline 更常用，因为大多数场景下我们关心的是"最多等多久"，而不是具体的截止时刻。

WithValue：传递请求级参数 ##

WithValue 在 Context 上绑定请求级别的参数，如 trace ID、user ID 等：

type contextKey string

const traceIDKey contextKey = "traceID"

func main() {
    ctx := context.WithValue(context.Background(), traceIDKey, "abc-123")

    processRequest(ctx)
}

func processRequest(ctx context.Context) {
    traceID := ctx.Value(traceIDKey).(string)
    fmt.Println("trace ID:", traceID)
}

注意：

  • Key 应该使用自定义类型（如 contextKey），而不是 string，避免不同包之间的 key 冲突。
  • WithValue 应该只用来传递请求级别的数据，不要把函数依赖、配置对象等挂到 Context 上。

完整示例：带超时的 HTTP 请求 ##

下面是一个完整的示例，整合了超时控制和 Context 传值的用法：

package main

import (
    "context"
    "fmt"
    "io/ioutil"
    "net/http"
    "time"
)

type contextKey string

func main() {
    // 创建带超时的 context，同时携带 traceID
    ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
    defer cancel()

    ctx = context.WithValue(ctx, contextKey("traceID"), "req-001")

    body, err := fetchURL(ctx, "https://httpbin.org/delay/2")
    if err != nil {
        fmt.Println("fetch failed:", err)
        return
    }
    fmt.Printf("response (%s): %s\n", ctx.Value(contextKey("traceID")), string(body)[:100])
}

func fetchURL(ctx context.Context, url string) ([]byte, error) {
    req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
    if err != nil {
        return nil, err
    }
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()
    return ioutil.ReadAll(resp.Body)
}

总结 ##

context 包虽然 API 很少，但它是 Go 并发编程中不可或缺的基础设施。几个关键原则：

  1. Context 是请求级别的，不要存到 struct 里长期持有，也不要当做全局变量。
  2. 总是传递 Context 作为函数的第一个参数，惯例命名为 ctx。
  3. defer cancel() 是好习惯，即使你不需要主动取消，它也能避免 goroutine 泄漏。
  4. WithValue 只传请求级元数据，不要用 Context 代替依赖注入。
